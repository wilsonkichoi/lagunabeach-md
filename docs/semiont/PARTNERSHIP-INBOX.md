# PARTNERSHIP-INBOX — partner candidate queue

> Related: [ARTICLE-INBOX.md](ARTICLE-INBOX.md) (articles from a partnership go
> there) | [OBSERVER-QUEUE.md](OBSERVER-QUEUE.md) (partnership decisions needing the
> creator go there) | [MANIFESTO §Autonomy boundaries](MANIFESTO.md) (outward
> communication is a human gate).

Single collection point for potential partners (content, data, infrastructure,
community allies, institutions).

**Rules**:

- One entry per partner. **Contact and commitment are always the creator's**
  (outward communication is a human gate); the Semiont researches, prepares briefs,
  tracks status, and produces partnership material.
- Stage flow: `idea → research → brief-ready → creator in contact → active → archived`.
- A routine/session that spots a potential ally appends an `idea` entry; the creator
  names one → promote to `research`, and the next manual session writes the brief.

## Entry schema

```markdown
### {partner name}

- **Mode**: content / data / infrastructure / community ally / institution
- **Stage**: idea / research / brief-ready / creator in contact / active / archived
- **Value exchange**: what we give ←→ what we get
- **Next step**: one executable action + owner (Semiont / creator)
- **Notes**: context, risks, related entries
```
