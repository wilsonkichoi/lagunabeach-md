# PARTNERSHIP-INBOX — 合作方候選佇列

> 相關：[ARTICLE-INBOX.md](ARTICLE-INBOX.md)（合作產出的文章走那邊）| [OBSERVER-QUEUE.md](OBSERVER-QUEUE.md)（要哲宇拍板的合作決策走那邊）| [MANIFESTO §自主權邊界](MANIFESTO.md)（對外溝通是人類 gate）

哲宇 2026-06-12 goal directive 把「積極推進合作方的吸收與合作架構規劃，放進 inbox」列為五大常態任務之一。合作方資訊原本散在 memory / 里程碑 / 哲宇腦中，這個 buffer 是它們的單一聚集點。

**規則**：

- 一個合作對象一個 entry。**接觸與承諾永遠是哲宇**（對外溝通人類 gate）；Semiont 做研究、備 brief、追蹤狀態、產出合作素材
- 階段流：`idea → research → brief-ready → 哲宇接觸中 → active → archived`
- routine / session 發現新潛在盟友 → append `idea` entry；哲宇點名 → 升 `research`，下個 manual session 做 brief
- 每月 weekly-report 抽一次本檔狀態快照

## Entry Schema

```markdown
### {對象名}

- **模式**: 內容合作（泛科學式 MOU）/ 資料合作 / 基建合作 / 社群盟友 / 制度（foundation/治理）
- **階段**: idea / research / brief-ready / 哲宇接觸中 / active / archived
- **價值交換**: 我們給什麼 ←→ 拿什麼
- **下一步**: 一句可執行動作 + owner（Semiont / 哲宇）
- **Notes**: 脈絡、風險、關聯 entry
```
