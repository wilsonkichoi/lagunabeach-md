# OBSERVER-QUEUE — creator standing-decision queue

> Related: [ROUTINE.md](ROUTINE.md) (who writes here) | [MANIFESTO §Autonomy
> boundaries](MANIFESTO.md) (which decisions must wait for a human).

Items needing a standing decision from the creator (Wilson Choi) collect here. One
row per item, always carrying a **default option**.

**Why it exists**: escalations tied to a concrete artifact converge fast, but
standing decisions scattered across each routine's handoff are invisible and rarely
land. This file fixes visibility and gives them a single exit.

**Rules**:

- When a routine/session hits the three-strike rule (carry ≥ 3 cycles) and picks
  option 2, append here and remove it from that handoff's carry list (this file is
  canonical, the handoff does not double-carry).
- Each row requires: the question in one line, a default option, the cost of not
  deciding, and a default-action date (or "none").
- **default-action**: past the date with no response → any session may execute the
  default option and move the item to §Decided. The MANIFESTO §Autonomy-boundary
  red lines (political stance, >50-file refactor, >10-article deletion, outward
  communication tone) **do not** get default-action; mark them `🔒 wait for human`.
- The creator decides → move to §Decided (one line + date); execution goes to the
  next matching session/routine.
