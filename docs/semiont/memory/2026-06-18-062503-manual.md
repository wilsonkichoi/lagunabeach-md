## 貢獻者排序 — 把 maintainer 往後放（fork-friendly）

文章 sidebar 的貢獻者順序 canonical 在 `scripts/core/build-git-info.mjs`：prebuild 掃 `git log`（newest-first），第一次見到誰就 push 誰。哲宇 merge 每個 PR 又跑 routine 的 frontmatter heal / babel re-sync，footprint 被維護主導，所以幾乎每篇多人文章他都排第一。`b9411d7b6` 加一道 `demoteMaintainer` stable pass 把他搬到最後、其他人維持原序。

哲宇問「有沒有更好的做法」時挖到真正的洞察：blanket demote 會 over-correct。`呂冠緯.md` 他寫了 88%、idlccp1984 改 1 行，demote 後 idlccp1984 反而被當成主作者。用 git numstat 量化證實了這個 over-credit。最後選的不是「按 diff 大小排」（要脆弱的 `--numstat -z` parsing、威脅 parity invariant），而是 demote 的對象從寫死的 `frank890417` 改成從 `git remote` 推導的 **repo owner** — identity-agnostic，fork（Sweden.md / Japan.md）自動 demote 自己的 owner。驗證 parity invariant 0 mismatch（revisionCount/lastModified byte-identical），3915 篇單人文章不動。

關鍵取捨：「最好的長期做法」不等於「最大的複雜度」。脆弱的 hot-path parsing 是長期 liability，robust + simple 才是。

## explore 熱門文章 + 共用 ArticleCard 縮圖卡 + dashboard 孢子可點

`0098950e4` 一個 commit 收三件：(1) dashboard 成效排行每列可點開新分頁到孢子貼文 — url 從 `spore-log.json` 經 `generate-dashboard-spores.py` 帶進 topPerformers，`dashboard-client.js` 整列 clickable + 標題留 anchor 供鍵盤/中鍵；(2) /explore 新增「熱門文章」section 讀 `dashboard-analytics.json` 的 `ga.topArticles7d`，lang-filter + 空資料自動隱藏；(3) 深度精選 + 熱門文章都改用既有的共用 `ArticleCard` `detailed` 變體，左圖右文 + per-article OG image 當縮圖。
