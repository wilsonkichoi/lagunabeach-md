## 重大根因修正 — 不是 hallucinated, 不是 rate-limit, 是 thumb size

初始假設「URLs 都 hallucinated」(基於 sample 2 條 新北市 URL `2017_Fort_San_Domingo` + `Lin_Family_Mansion_and_Garden_2` API verify 都 MISSING) 被後續 API batch audit 推翻：

- 連江縣: 5/5 real ✅
- 新竹縣: 5/5 real ✅
- 澎湖縣: 4/5 real (1 hallucinated)
- 嘉義縣: 1/1 real + 1 hallucinated (rest API error)
- 苗栗縣: 3/5 real
- 其他 12+ 縣市: API audit hit rate limit (0.4s sleep 太快, 66 ERROR no result)

可確認 partial sample：**~3% hallucinated, ~97% real but 1280px thumb 不存在**。cache-county-images.mjs 預設 1280px thumb URL 對原圖小於 1280 寬的會 404 (Wikimedia 只生成 ≤ 原圖寬的 thumb)。後 fallback original URL 撞 429 wall + 5 attempts × 5s backoff 不夠。

5/18 教訓「88 圖留給 cron 慢慢補」隱含「cron 會自然恢復」假設**錯誤**——cron 用同一個工具同一個 thumb 假設, 4 天後仍 stuck. 真正 fix 是 multi-size variants + 真 fallback chain (Wave 1+2+3 retrospective surface 出的 wiki-fetch.py 5-strategy stack).
