## 觸發

哲宇 `/twmd-become` 帶一個 idea：每篇相關文章製作時調查相關資料集，讓文章 SSODT 包含指向 Twinkle Hub 資料集的指標跟「為什麼指向」，要求深度研究抓幾個主題思考。之後三波加碼：「yes 直接 pilot、文章端可以出現 Twinkle Hub（週五對談 demo）」、`/goal` 要求數據 subnav 加 OpenData 策展頁（完整盤點 + 分析組合 + 多語機制）、最後「幫我把部署一起全部修完」。

## 研究與六篇 pilot

實測戳破三件事：Twinkle Hub 的 API 在 5/11 之後整個改版（強制 MCP session 握手、40 工具重組為 21 個純資料層、twtools 確定性工具下架、query_rows 第一發 429）；5/11 卡住 Layer 2b 的 search 缺陷已自癒（「咖啡」0→3 hits，六個主題全命中白金級資料集）；dataset_id 直接對應 `data.gov.tw/dataset/{id}` 持久頁。三件事合成雙層指標設計：data.gov.tw 是身分、Twinkle Hub 是查詢加值層，指標寫成靜態 metadata 不在執行期依賴 alpha API。研究報告與修好的 wrapper（補 429 退避）以 `08567837d` 落檔，pilot 以 `7678a430b` ship：核能、國宅、健保、夜市、交通、九合一共 15 個指標，frontmatter `datasets:` 給 AI 讀者、文末「公開數據」段給人類，策展句紀律是「遮掉資料集名稱還猜得出接文章哪一段」。

## 部署解封（架構解）

build 在 internal-links 閘門紅燈（gated 6.67% ≥ 2%）。診斷出三個每頁複製的結構源，合計約 1.9 萬條死鏈：語言切換器對非文章頁假設「所有語言都存在」（650 個 semiont 子頁每頁 4-5 條）；nav 的 translatePath 盲目加語言前綴（/semiont 的舊修復在某次重構中遺失、/bench 對 es/fr 裸奔）；RelatedArticleCard 呼叫端沒傳 lang（非中文文章頁的相關文章卡全生無前綴連結，被 verifier 按 target 前綴記到 zh-TW 桶，即詭異的 12%）。解法是 `f8b2893bc` 新增 staticRoutes.ts：以 src/pages 樹為靜態路由 SSOT 做存在性查詢，切換器 has 與 navHref 都吃它，新語言新頁出生自動正確。結果 0.56%，全閘門過。時序釐清：早上撞到的 2% 閘門是平行 audit session 的中間狀態（量測基底錯，後定案 7.0 + step-down），CF 端 11:07 的 push 其實過了，pilot 文章在 heal 前就已上線；heal 的價值在砍掉真實存在的 1.9 萬條死鏈，跟閘門數字無關。
