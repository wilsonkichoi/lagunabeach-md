## Lessons (今天的 finale 新增)

### Lesson #1：Agent 必須有 hard search limit

桃園市 Sonnet agent 第一次跑 6 小時無活動 deadlock — 沒有上限 instruction 時，agent 可能進入「研究 → 更想研究 → 更想研究」遞迴 cascade。**fix**：所有 Sonnet Stage 1 prompt 加 `Hard limit: 45 次搜尋上限，到了立刻停止 search 進寫作`。pattern 跟 v1.6 SQUEEZE batch failure（cascade with no exit）isomorphic — pattern matching 已內化進下 batch prompts。

### Lesson #2：cron `twmd-data-refresh-am` 會 grab in-flight files

Sonnet agent 寫到 main 絕對路徑時，同 repo 同步在跑的 cron 看到 dirty tree → 一起 commit。Batch 4 三隻 agent Stage 1 被 cron `4ad0aa420` 帶進去 commit，主 session 完全沒控制。**fix 方向**：cron 開始 sync 前先 `git status` check 是否有未預期 working tree 變動 + 主 session spawn agent 期間考慮暫停特定 cron。Post-script memory `5a8659f8e` 已記錄但 fix 尚未實作 — 留給未來。

### Lesson #3：Opus agent 自己會抓 task spec 錯誤

台中市 Opus agent 寫的時候自己 verify「大甲遶境跨 5 縣市」是錯誤的（正確跨 4 縣市：台中 → 彰化 → 雲林 → 嘉義），直接寫正確版本。**fix**：Opus prompt 不要直接把「verified facts」當絕對真理塞進去 — 留 escape hatch 讓 agent 自己 verify。Lesson 跟 batch 2 emergent three-layer fact-check 同源 — main 預設可能錯，agent 應該有 authority 校正。

### Lesson #4：22 縣市差異反 over-pattern-match

Pilot 基隆 retrospective 5 條校準 + Sonnet/Opus 平行 spawn pattern 跑 5 個 batch 成功 — 但每個城市 fault line 不同，prompt 必須容納差異：
- 直轄市（台北/新北/桃園/台中/台南/高雄）需要「平等不歧視 + 結構性事實」處理（避免「天龍國反作用」）
- 縣（金門/連江/澎湖/宜蘭）需要 isolation 結構（島 + 戰地）
- 山地縣（南投/花蓮/台東）需要原住民族 + 921 / 莫拉克災難 framing
- 工業縣（雲林/彰化）需要環境正義 + 衰退 frame 具體數字
- 半導體城（新竹市）+ 重工業港都（高雄）需要技術產業史
每篇都用「pilot pattern」會 over-fit。Pattern 是骨架不是模子。
