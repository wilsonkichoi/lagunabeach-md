## wiki-fetch.py 兩個結構性 issue retrospective

3 個 Wave 2 agents（寶藏巖 / 公館 / 大龍峒）獨立報告 wiki-fetch.py 限制：

**Latin-1 encoding bug on CJK URLs** — `urllib.request` 在 build Referer header 時對 CJK 沒做 percent-encoding → 報 latin-1 codec error → fetch fail。應該在 `_fetch_once()` 把 referer string 過 `urllib.parse.quote(referer, safe=':/')` encode。

**Concurrency lock 變 serial bottleneck** — 9 個 parallel agent 同時搶用 `/tmp/.wiki-fetch.lock` 變 serial 等候。設計 `MIN_INTERVAL_S = 2.0` 跨 process 單一 lock 適合 single-agent batch 不適合 high-concurrency parallel dispatch。應該改 N-slot semaphore 或 per-host concurrent limit（3 per WMF policy）。

Agents 都 improvise 用直 curl + Referer Commons File: URL + thumb endpoint variant (`/wikipedia/commons/thumb/.../1280px-{name}`) 都成功 cache，部分用 Wayback Machine fallback（中山北路條通 agent 3 個圖）也工作。Tool 設計方向對，兩個 bug 要修。LESSONS-INBOX 候選 2 條已寫進 Wave 2 retrospective entry in ARTICLE-DONE-LOG。
