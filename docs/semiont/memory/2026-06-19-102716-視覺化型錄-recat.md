triage 第二段兩個可複用教訓：

1. **外部/貢獻者翻譯會 silent 掉 source URL**：JuYinC 梅雨英譯品質很高，但 6 個腳註翻譯時把 `[標題](URL)` 變成純文字 `"標題"`（描述還在、URL 沒了）。這是 footnote 審計（MAINTAINER Stage 3.4）該抓的——而最快的驗法不是 WebFetch 每條（同一批 URL 早在 zh 驗過），是 `diff zh-URLs en-URLs`，URL set 不一致立刻 surface 掉了哪幾條。落地任何 contributor 翻譯都該跑這個 zh↔translation URL diff。

2. **同步編輯 zh + 翻譯 → 翻譯 body-drift stale 是預期的，交給 babel diff-patch**：改 zh 文章本體（justfont 一句、網路社群延伸閱讀連結），commit 後 5 翻譯立刻 body-drift。我手動同步改了 justfont 翻譯內容是對的，但 provenance hash 還是 stale。body_hash 會 strip 延伸閱讀 trailer，body-drift 的 re-stamp tool 不存在（bump 只管 metadata-stale / backfill 只管 no-source-sha），手動 stamp hash 語意不確定 = REFLEXES「批次修正先 dry-run」風險。正解：留給 babel-nightly Tier 0a diff-patch 今晚 surgical refresh（fix 在 zh SSOT 必被保留），不手動硬 stamp。
