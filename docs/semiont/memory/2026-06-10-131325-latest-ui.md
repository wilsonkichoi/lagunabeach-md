## ArticleCard 統一與 detailed 卡演化

關鍵發現是 `ArticleCard.astro` 早就宣告 `density="row"` 但全站零 consumer，而 /latest 手刻著自己的列。讓 /latest 的設計成為 row 本體，繞開元件統一的 prop 數紅線。第二輪升級成 explore featured 式 detailed 卡（框 + 完整描述 + tags + 引用/閱讀 footer），順手把 `buildIndex` 補了 readingTime / tags / footnotes（同迴圈讀檔零成本）。盤點出中文文章只有 135/786（17%）有 frontmatter image，這個數字決定了後面所有圖片設計的方向，也開了補圖 spawn chip。`cdc831d61` ship。

## 左側時間軸與 timeline 共用元件

哲宇點名參考 changelog 的做法，/latest 從月分組平鋪重寫成日分組時間軸（dot + 連接線 + 日標題 + 篇數 badge + sticky date-nav 跳轉欄 + IntersectionObserver 高亮）。下一輪再把骨架抽成 `components/timeline/`（TimelineDay + TimelineDayNav，emerald/blue 雙主題），/changelog 與 CommitLog timeline variant 遷到同一具骨架，兩頁的 filter/搜尋/badge 腳本只換 selector 零行為變更。`f5039d8b8` ship。

哲宇 callout「同一天內也要最新在前」，驗證後發現排序其實一直正確（6/07 抽查 21:00→16:52 嚴格降序），感知差異來自中午 batch heal 造成六篇同秒 timestamp 的 tie。comparator 仍升級成 epoch-based 防未來 Z/+08:00 混格式的字典序陷阱。先驗證再修，這次省掉一個不存在的 bug。
