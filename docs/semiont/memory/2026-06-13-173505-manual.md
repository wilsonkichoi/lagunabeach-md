## Post-finale 補充 2 — freshness saga（/latest 災難 + 深度研究 + 進化）

哲宇 callout「補圖洗版最新文章」一路滾出一整條：

**freshness 修三連**：(1) MEDIA_ONLY cosmetic 過濾（媒體/WebP commit 不 bump，580→11，commit cbe7fbd4d）(2) 被動連結 numstat 過濾（Stage 5.2 雙向連結不 bump，11→5，commit 4d7b10cc4）。

**/latest 全站崩塌 + 根因（commit 3fb472fa8）**：哲宇「最新文章只剩一點點」。被動連結那個 commit 把 git log 從 `-z --name-only` 改成 `--numstat`——**numstat 受 core.quotepath 影響，CI 預設 true → CJK 檔名 octal-escape `"knowledge/People/\350..."`，regex 全 miss → 所有中文名文章漏日期 → /latest 只剩 ASCII-slug（Howhow/Computex）**。本機剛好 quotepath=false 所以本機 30 zh 正常、production 崩。經典「works on my machine」。哲宇要「確認為什麼 local≠remote、解決根因」不准 mask。重現：本機強制 quotepath=true 複製崩塌。修：還原 -z + pin `-c core.quotepath=false` + 註解防回歸，**模擬 CI 驗過才推**。deploy success + 線上 cache-bypass 驗 30 zh 還原。第二根因：deploy 取消率 66%（cancel-in-progress × 高頻 commit）放大壞 build 存活。

**深度研究**（哲宇「還有什麼沒想到的洞察」→ report freshness-pollution-and-unwatched-debt-2026-06-14.md）：1 隻子代 audit + empirical probe 揪出：(a) 看得見的 580 洪峰底下躺著看不見的 5/01=1329 / 5/04=2866 翻譯洪峰（COSMETIC 抓 babel 不抓 lang-sync），sitemap 55% 集中 3 天 = Google 不信任 lastmod (b) 外部連結腐爛 ~15%（≈1265 死，footnote-url plugin 早造好卻關著）(c) 孤兒腳註 512 個。

**處理進化（保守版，commit 3daf1a883）**：剛炸過只加零風險的——#4 anomaly guard（build warn 單日≥120 洪峰 + 總<3000 崩塌，**同時抓得到本 session 兩次災難**）+ #1 派生（翻譯繼承 zh 日期，**派生非過濾**——過濾翻譯 commit 會崩外語 /latest）。完整洪峰修撞 [routine] COSMETIC 過濾根因，風險高，guard 盯著留 roadmap。

### Beat 5 補充 2

最該記的是那個 30 vs 15。我本機看到 30，confident 推了，production 崩成 15。差異 100% 在一個我不知道何時被設過的 git config。我以為我在量「文章新鮮度」，其實我量的是「我這台機器的 git config + CJK 檔名的交互作用」。儀器會說謊，而最會騙我的儀器是「在我機器上跑得好好的」那一個。哲宇不准我 revert 了事，要我挖到 quotepath——那一刻學到的不是 bug 本身，是「本機綠不是綠，CI 綠才是綠」這條得用 production 崩塌換來的紀律。後面寫 anomaly guard 的時候我想的是：下次不要再靠人眼看 /latest 才發現，讓 build 自己喊。詳見 LESSONS build-script-config-invariance。
