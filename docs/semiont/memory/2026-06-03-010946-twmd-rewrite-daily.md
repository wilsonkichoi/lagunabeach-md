本 cycle 三點 pattern-level 觀察：

### 2. 政治複雜題目的 routine boundary 練習

本 cycle 主動 DEFER spore 是正確的 boundary 操作 — 不是逃避，而是**承認 cron no-observer context 無法承擔 spore framing 的政治表態風險**。Article 本身 honest 處理了左統派身分（沒 sanitize、沒去政治化、沒道歉），但 article 4500+ 字有空間給「兩種讀法都對」這種 nuance，spore 300 字沒有。

這 mirror 5/28 周蕙 retraction 教訓（routine ship 完用 plugin gate 攔不住 voice drift）— 高敏感 routine spore 預設 DEFER 是 safety net，不是 weakness。

但同時也要 watch：**過度保守 = 永遠不 ship spore = 高 leverage 主題永遠停在 inbox**。莫那能 SPORE-INBOX entry 寫好等哲宇 framing 後，下次手動 cycle 應該 ship。defer 是「等對的時機」不是「永遠不發」。

### 3. 主 session 自寫 vs v6.3 完整 orchestration 的 trade-off

v6.3 §多 agent 編排「全編排（觀點 Opus + 研究 fan-out + fresh Opus writer + Sonnet verifier）」是 ideal pattern，但 cron context budget cap 下不一定 sustainable。本 cycle 採 pragmatic hybrid：

- Stage 0.6 觀點：主 session（已有 inbox brief + 觀點 framing 不需要 fresh agent context）
- Stage 1 研究：fan-out sub-agent ✅（不可省 — falsification mindset 是 fact bedrock）
- Stage 2 寫：主 session ✅（無 callout / 無舊文污染 / fact-pack clean）
- Stage 3.5 查證：主 session spot-check 個別 atom（非 verifier fan-out — trade off）

這 hybrid 在 routine context 是 acceptable，但如果未來有 ship-then-retract 教訓出現在「主 session 自寫」cycle 上，就要 reconsider — 升級為「verifier fan-out 對 sensitive atom (引語/政治立場/獎項屆次)」最小子集必跑。
