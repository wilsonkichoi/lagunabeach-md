## Google 新聞 + 整站 SEO 深度研究

Google 新聞問題拆兩層：技術上大致合格（2024-25 起免申請、自動抓取），但內容會不會真出現＝95% 不會，因為 evergreen 知識庫不是新聞（官方原話「Simply put, no.」）。真正風險是 scaled-content-abuse（2026/3 頭號打擊），而 Taiwan.md 每天多篇 AI 文的輪廓恰好像它要獵殺的對象——辯護靠的正是 Google 列的豁免（編輯把關 + 原創價值 + 透明）。建議別追 Google 新聞，真正適配是 Google Discover + 已在做的 AI 引用。

整站 SEO audit 用真實 SC 數據：110,430 曝光 / **1.39% CTR**——基本功頂標（description 99% / alt 100% / 結構化資料齊），問題在「被看見了沒人點」。`computex` 排名 1.23 卻 0.94% CTR。

接著 5-agent 平行深度研究（primary source 優先 + falsification），結果**推翻自己前一輪 audit 兩項建議**：FAQPage 2026-05-07 Google 對全站廢除（我建議的是死方案）；trailingSlash 在 CF Pages 已被 308 自動解決（near no-op）。Person schema 無 rich result、reframe 成 entity/AI 引用。報告 `reports/seo-optimization-plan-2026-06-07.md`（commit `0461d7882`）逐項給研究發現 + 設計 + 風險 + 驗收。

## freshness 完整修復 + 儀器化（commit `c1403e259`）

5 項裡 P0 freshness 是真贏，且比想像嚴重：sitemap 全域 `lastmod: new Date()` 是 Google 明文點名反模式（→ 對整站 lastmod 停止信任）；Article dateModified 永遠＝datePublished＝EVOLVE 白做。

修法走儀器化：`scripts/core/build-content-dates.mjs` 一次 git pass 算每篇 last-meaningful-commit，cosmetic 過濾 babel/routine/lint 避免假 fresh（5359 URL，skip 7206 cosmetic touch），寫 `content-dates.json`，sitemap serialize + article.template JSON-LD 同源讀取。測試：astro build exit 0、黃山料頁 JSON-LD dateModified=真實 git 日期、sitemap 模擬 81% 真實日期 0% fake（修前 100% fake today）。

過程兩個 catch 值得記：(1) 第一次 build 後發現 773 個中文 slug 頁 sitemap 全無 lastmod——`new URL().pathname` 回 percent-encoded，content-dates key 是 raw UTF-8，`decodeURIComponent` 修好（模擬 0→780 matched）；(2) pre-commit hook 擋下我的 hardcoded 語言陣列（REFLEXES #20），改從 registry derive——免疫系統正常運作。
