## 視覺體檢：v1.0 的圖表在 production 壞了六天沒人看見

用 Playwright 對型錄頁逐模組截圖（17 模組 × light/dark/mobile = 51 張，preview MCP 的截圖在程式化捲動後會回白屏，改用 repo 既有的 playwright 寫一次性 capture script）。抓到 11 個問題，最重的三個是 cascade leak：tw-quote 的 blockquote 被 `.prose blockquote` 的灰框＋斜體蓋走變成雙重框；tw-heatmap 的字色被 `.prose td` 蓋掉，深色格在兩個模式都接近不可讀；figure/versus/timeline 三個模組的 `來源：` 列被共通抽取邏輯吃掉之後從來沒渲染出來。這些從 6/06 v1.0 ship 當天就壞著——當時驗證方式是 curl 確認 markup 存在，沒有看像素。修法：模板 prose 規則加 `:not(.tw-quote-text)` / `:not(.tw-heatmap-table)` 排除 + 模組 CSS 顯式歸零。

另外六個設計層問題一起修：heatmap 冷暖雙色相在中段混成泥色（改單色相淺→深，`--tw-heat-lo/hi` 雙主題）；tw-line 的 SVG 文字跟著容器放大到 ~25px（viewBox 400 + max-width 520 置中）；多序列面積互疊（面積只給單序列）；bars 不支援負值（`_num` 保負號 + 分歧條模式）；`.tw-line-s0/1/2` 死 CSS（改 class 驅動，第一輪截圖立刻抓到我把 fill 掛上 path 讓折線自己填成色塊，第二輪修正）；基準線無支援（`基準：標籤 | 值` 列 → 虛線無點）。

## 缺口分析 + 7 個新模組

兩個背景 research agent 並行：一線掃 FT Visual Vocabulary 九大類＋Datawrapper 使用統計＋DataViz Catalogue 全集，一線掃 The Pudding／報導者／OWID／NYT 的閱讀體驗 pattern。對照下來 v1.0 蓋住 5/9 類，缺分布、空間、變異、量級人性化。新模組七個全部 ship：`tw-slope`（兩點斜率）、`tw-dot`（共用軸 dot strip/range）、`tw-stack`（100% 跨列組成）、`tw-pyramid`（背對背）、`tw-tiles`（**台灣 22 縣市磚圖**）、`tw-iso`（單位圖）、`tw-note`（【說明】(註) 方法盒）。共通約定同步統一：17 模組全收 `來源：` 列、資料模組第一列無 `|` 即標題、bars/slope/dot 支援 `*` 強調退灰。

tw-tiles 是這輪最重要的器官：22 縣市佈局寫死在 renderer，不畫台灣輪廓，所以對「橄欖馬鈴薯」形狀幻覺結構性免疫（REFLEXES #61 的架構解）。臺/台與市縣字尾自動正規化；翻譯版縣市名對不上 60% 時整模組退化成排序 bars，內容不消失。型錄 demo 直接用 22縣市文已驗證的戶政司高齡化全集，22/22 對應成功。
