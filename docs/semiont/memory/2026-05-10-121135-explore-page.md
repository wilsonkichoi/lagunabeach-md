## ROUTINE v1.1 — maintainer 1d 2x + PR backlog SSOT 收割者

PR [#985](https://github.com/frank890417/taiwan-md/pull/985) → squash-merged as `81f120ee5`。哲宇觀察到 cron routine 開的 PR（[#983](https://github.com/frank890417/taiwan-md/pull/983) self-evolve / [#976](https://github.com/frank890417/taiwan-md/pull/976) maintainer memory 等）累積成 open backlog 沒人 merge。診斷出來是兩個結構性問題：每條 routine 各自跑 `gh pr merge` 等於 6 處複寫 hard-gate 邏輯（drift 風險違反 §指標 over 複寫）；routine 跑完不可能等 CI 5-10 min 才 merge — 等 = 浪費 routine 時間，不等 = 賭 CI 過。

兩條結構性修正：**A.** maintainer 從 1d 一次升級為 1d 兩次，taskId `twmd-maintainer-daily` 留作 AM slot（scheduled-tasks 不支援 rename），新建 `twmd-maintainer-pm` 跑 cron `7 21 * * *`。**B.** §collect-and-merge SOP 新增：對每個 open PR 走 hard-gate 分流 — `🧬 [routine]` prefix + author=owner + CI green + mergeable + age ≥ 5 min → squash merge；CI pending → defer 下個 cycle；CI fail / conflict → 留 open + LESSONS entry；contributor / observer PR → 永不 auto-merge。其他 routine 開 PR 即收工，集中由 maintainer 收割。

ROUTINE.md v1.0 → v1.1：排程表 9→10 條、§設計原則加「集中收割比分散收割可控」5 個理由、Stage 4 ship policy 收緊（只有 maintainer 可以 auto-merge 自己 PR）、§失敗 escalation 加「routine PR backlog > 5 = 紅燈」。同步 update `twmd-maintainer-daily` SKILL.md prompt + create `twmd-maintainer-pm` SKILL.md（兩個檔案在 `~/.claude/scheduled-tasks/` 不在 repo per ROUTINE §同步來源）。
