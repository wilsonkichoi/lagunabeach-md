## /latest 手機版三修

三處改動都在 `/latest`，commit `0b24cfcee` push main（3 檔 / +13 -6）。

**圖上文下**改 [ArticleCard.astro](../../../src/components/ArticleCard.astro) detailed 變體的手機 breakpoint（`max-[768px]`）：卡片轉 `flex-direction: column`、縮圖 `order:-1` 拉到卡頂、`width:100%` + `aspect-ratio:16/10` 做滿版橫幅，桌機版圖右文左不動。這裡有個坑：縮圖在 DOM 排在 `.article-card-body` 之後，純 `flex-direction: column` 會變成文上圖下（剛好相反方向）。Explore agent 探路時給的就是純 column 的 fix，會做錯；讀元件 source 確認 DOM 子節點順序後改用 `order:-1` 才對（REFLEXES #16 agent 是線索不是 source 的又一次驗證）。

**時間軸線可見度**：哲宇說「線斷的顏色看不到」。量了才知道不是斷——[TimelineDay.astro](../../../src/components/timeline/TimelineDay.astro) 的 emerald 連接線是 `#d1fae5`（emerald-100，白底上 RGB 209,250,229 近乎透明），整條線一直都在，只是看不見。加深到 emerald-300 `#6ee7b7`，blue accent 同步 `#dbeafe→#93c5fd`（同一具共用骨架 /changelog 也吃 blue）。

**hero/filter 空白**：哲宇說「奇怪的空白」。量到是 [latest.template.astro](../../../src/templates/latest.template.astro) 內容容器的 `pt-10`/`max-[768px]:pt-6`（桌機 40 / 手機 24px）露出透明底色變一條灰帶。移掉 pt 後 sticky filter 工具列貼齊 hero 下緣，灰帶消失。

一個觀察：三修裡有兩修（看不見的線、尷尬的 gap）是我自己的圖上文下改動把卡片變高、像放大鏡一樣照出來的潛伏 CSS 債。本來矮卡片時沒人注意。
