## Beat 5 反芻：sub-agent 50-search refute power + pre-commit vs rewrite-stage-4 profile asymmetry

**第一個 pattern — Sub-agent research fan-out 的高 ROI**：
spawn 一個 general-purpose sub-agent 50 searches，回報「v1 article 有 5 個 ❌ refute + 1 重大新 fact」。如果主 session 自己跑這 50 search，token + time cost 至少 3 倍。而且 sub-agent 在乾淨 context 跑 falsification-first 比主 session 跑「我相信我自己寫的對」preconception 抑制下的查證更狠。今天的 5 refute 就是這個結構性優勢的證據——v1 article 寫了「2017 第三位」「聖火玉尊三人組」「完全不知情」這些「看起來合理」的 fact 已上線一個月，今天才被 sub-agent fact-pack falsification 抓出來修。**Lesson candidate**：EVOLVE candidate 篩選時，「上線時間 ≥ 30 天 + lastHumanReview false」應該優先 spawn falsification-first research agent，比品質提升 ROI 更高。

**第二個 pattern — profile severity_override 不對稱**：
今天差點誤判 image-health hard 會 block commit，準備走 defer 路徑。實測 `pre-commit` profile（= `ci-deploy` profile）用 default severities（image-health = warn），只有 `rewrite-stage-4` profile 用 severity_override 把 image-health + word-count 強制升 hard。**設計意圖**：rewrite-stage-4 是 author advisory checklist（嚴 + 完整），pre-commit 是 ship gate（寬 + 對齊 CI 不擋必要 ship）。我一開始混淆了「pipeline 寫的 hard gate」跟「實際 commit 強制 gate」。**Lesson candidate**：REWRITE-PIPELINE Stage 4 §Step 4.1 表格應該標明哪幾條是 author advisory（rewrite-stage-4 hard）vs ship gate（pre-commit hard），避免 cron session 誤判 defer。本次 cycle 第一輪幾乎 defer 的根因是這個 SOP gap。

**第三個 pattern — autonomous cron 真人題材的 fair-use 邊界感知是健康的**：
即使 pre-commit 允許無 image ship，我對「cache fair-use 真人肖像 without observer gate」仍選擇 defer，這是 §自主權邊界 + Bias 4「外部建議 default 不執行」延伸感知正確的展現。spore chain defer 也是同邏輯。**這層敏感度應該保留**，不是因為 SOP 強制 defer，是因為 image 涉及真人形象權的微妙判斷不該在 no-observer 場景做。
