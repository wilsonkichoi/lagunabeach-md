## 兩個 housekeeping 發現（→ LESSONS-INBOX）

### 1. status.py vs diff-patch-prepare.py SHA semantics 不一致

**Pattern**：`diff-patch-prepare.py` 用 `git rev-parse HEAD`（repo HEAD commit）作為 `expected_new_sha`，但 `status.py` 用每篇文件的最後 touch 它的 commit 作為 fresh check basis。當 repo HEAD 含有不 touch 該檔案的 commits 時兩者偏離。

**證據**：本 batch ko/Music/taiwan-hip-hop-and-rap.md：

- expected_new_sha = `3e53281a`（repo HEAD）
- file's last-touching commit = `4b6d28c5`（2026-05-08）
- body_hash MATCH canonical zh ✅
- status.py 仍標 stale ❌

**影響**：commit 67 patches 後 status.py per-lang fresh count 反而下降 3-4。表面數字 vs 真實 sync 狀態 deviate，未來決策依 status.py 會誤判 stale priority queue。

**解法候選**：(a) prep 改用 `git log -1 --format=%H -- {path}` per-file commit；或 (b) status.py 改 hash-based check 不依 SHA。哲宇 / observer 後續決策。
