## tw-versus 手機修 + 無名小站內容增補

`tw-versus` 的根因單純：手機 ≤560px 的 media rule 只把兩欄 grid 壓成單欄（`grid-template-columns:1fr`），於是「無名 3.9 分 vs FB 6.3 分」這種成對對比被拉直成一串獨立 item，每格只剩一條色邊當線索，表頭一滑掉就看不出誰對誰，把二維對比硬攤平成一維是結構性丟失配對。修法是 renderer 每格內嵌側別標籤 span（桌機 `display:none`、手機顯示），CSS 手機把每列收成「成對小卡」（卡片邊框分組 + 行內橘/藍標籤）。桌機兩欄不動，全站 tw-versus 共用此 renderer（`b29977072`）。graph.md §七 的像素閘門過 light-mobile / desktop / dark-mobile 三變體才算驗完。

子宮→搖籃跟圖片增補打包進 `860aea532`。用詞上哲宇先丟「子宮→孵化器」又反悔說中文少人這樣用，給的三個連結（agirls / nownews / iThome）其實都沒用搖籃類比喻、且圖全是版權 UGC 不能 cache，所以連結沒解決用詞、也沒給圖。AskUserQuestion 後哲宇選「搖籃」，描述加內文兩處改掉，並錯開既有 30 秒概覽的用字；Fun Party 圖說的「搖籃」改「舞台」消同段重複。圖片他選「只放真實照片、不用 logo 湊數」保 S 級策展純度，Commons 上真無名照片基本只有現有 2 張，所以補的是時代脈絡照（交大光復校區 / Yahoo Kimo 2008 / PTT BBS 進站畫面），分置創站·BBS·併購三節點，2→5。值得記的是哲宇口中的「8+」其實就是 health gate 的 `media-richness` advisory（長文圖+影片≥8），他知情後選 5 張封頂、把完整解（補 video iframe）留待下次。
