## 根因：dev/prod 大小寫不對稱

Astro 路由用小寫 `categorySlug` 產出（`getStaticPaths` → `/en/politics/`），但 `generate-api.js` 的 `generateArticleUrl` 沿用磁碟 PascalCase 資料夾名（`/en/Politics/`）。

- **macOS 開發機檔案系統大小寫不敏感**，`/en/Politics/` 自動接到 `/en/politics/`，本機完全看不到 bug。
- **Cloudflare Pages 生產環境大小寫敏感**，`/en/Politics/` 直接 404。這就是 bug 只在生產出現、躲過所有本機測試的原因。
- zh-TW 沒事純屬巧合：舊 client regex `replace(/^\/([^\/]+)/, lowercase)` 只小寫「第一段」，zh-TW 第一段剛好是類別；非預設語言第一段是語言碼，類別段沒被碰 → 全 404。

## 修法（架構解 > 守備修補）

1. **根**：`generateArticleUrl` 改小寫「類別段」（倒數第二段，`[lang/]Category/slug` 通用），articles.json url 變 canonical（4777 筆 0 大寫類別）。
2. **縱深**：兩個 client consumer 改小寫倒數第二段，擋 CDN 快取過渡窗口的舊 JSON。
3. **第二類殘留 404**：驗證時抓到 `resources/*`（`/resources` 單頁資料、無 `[category]/[slug]` 路由）+ 無類別孤兒 `en/taiwan-semiconductor-industry.md` 漏進隨機池。加**路由類別閘門**（`categoryList` SSOT，三點同源）擋掉。

articles.json 是 build artifact（prebuild 重生），不入 commit。Commit `034f76a2b`（3 source files）。

## 結構性教訓（LESSONS 候選）

**case-sensitivity 是 dev/prod 隱形斷層，驗證必須在 case-sensitive 表面做。** 我第一版想用 localhost HEAD 檢查隨機 URL 是否 200 — 但 dev server 跑在 macOS（大小寫不敏感），`/en/Politics/` 會假性 200，正好複製出 bug 隱身的條件。改用對 `dist/` 逐段 `os.listdir` 精確大小寫成員檢查（en/ja/ko/zh 共 3177 池 0 404）才是有效證明。**規則：任何路由 / 連結正確性驗證，不能信 macOS 本機檔案系統與 localhost；要嘛對 dist 逐段精確大小寫比對，要嘛在 case-sensitive 環境測。** 對應 REFLEXES #24（工具在說謊：localhost 200 是假 PASS）+ #67（已驗過要帶證據）。

第二條：**衍生 artifact 的 url 欄位若本身就 404，是 latent landmine。** articles.json 的 url 對所有語言都非 canonical，只是消費者剛好半補了 zh。`url` 欄位該是「能到的網址」，不是磁碟路徑的字面投影。修在生成端讓全站消費者一次受惠。
