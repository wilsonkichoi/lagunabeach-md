## Beat 2 — 進化（在執行中發現的 pipeline gap）

### Pipeline gap：multi-paragraph 中文 spore 的 paste path 是 broken 的

走 SOCIAL-POSTING-PIPELINE v0.6 工作流時連續撞 4 個 layer 的 entropy：

1. **Chrome MCP `key cmd+v`** 需要 OS-level keyboard event — Chrome MCP 沒 `key` tool（只有 `computer.key`）
2. **computer-use tier**：Chrome 在 computer-use granted at "read" tier，無法 keystroke 直接 send（必須走 Chrome MCP）
3. **Chrome MCP `computer.key`** 可以！原 SOCIAL-POSTING-PIPELINE 沒明確標 `mcp__Claude_in_Chrome__computer` 提供 `key` action — pipeline ✅
4. **execCommand insertText with `\n\n`** 在 X ProseMirror → collapse 成單塊 prose + URL auto-link 6× duplication 災難
5. **paragraph-by-paragraph insertText + key enter enter** between → 跑了一次但 cmd+v 第二次撈到別的 clipboard content 結果亂掉（哲宇 mid-callout 指出剪貼簿被取代）
6. **`cat <<EOF | pbcopy` 中文 UTF-8** → Mac Chinese locale 強制 Big5 轉碼，pbpaste 讀回 0 bytes / 顯示 Big5 garbage
7. **osascript `read POSIX file as utf8` + set clipboard** → AppleScript runtime 把 NSString 轉系統預設編碼（Big5）寫 clipboard，結果不是 UTF-8
8. **JXA `NSPasteboard.setStringForType(nsstr, 'public.utf8-plain-text')`** → ✅ 真的寫 UTF-8 到 clipboard，Chrome paste handler 正確讀到，paragraph breaks 正確 render

**核心發現**：JXA NSPasteboard 是 Mac Chinese locale 環境下 multi-paragraph 中文 paste 的唯一 reliable path。pbcopy / osascript classic / execCommand 都各有 corruption mode。

### 額外發現：X paste handler 是「智能 prose splitter」

當 clipboard 含 `\n\n` 分隔的 multi-paragraph 文字 → X compose 的 paste handler 自動 split 成 ProseMirror 段落 block，帶 visual blank line spacing。但 execCommand insertText 走的是不同 path（直接 insert 為 single-block text node）— paste path 跟 insertText path 對 ProseMirror 行為**根本不同**。

### 為什麼 4 層 gate 而不是 1 層

5/28 22:30 撞牆撞了 6 種 paste path 才找到 JXA 的關鍵 lesson：**單一 layer instrument 化會 leak**。

- Plugin Rule #14.A/B 只跑在 **blueprint file** — 不跑在 **compose textbox** 也不跑在 **rendered output**
- Pre-ship check 7+8 跑在 **compose textbox** — 不跑在 **blueprint file**
- Post-ship check 6 跑在 **rendered output** — 是 caching layer drift 的最後一道網

三個物理層（file / compose / rendered）有各自的 silent failure mode：

- **File 層**：blueprint 寫對了不代表會被 reliably paste 到 compose（execCommand 路徑 collapse / pbcopy Big5 corrupt）
- **Compose 層**：compose 看起來分段不代表 ship 後 rendered 也分段（X/Threads paste handler vs render layer 有 caching drift possibility）
- **Rendered 層**：rendered output 是真實讀者看到的東西 — 最終 ground truth

對應 MANIFESTO §架構解 vs 守備修補：「6 次 retry 找 SOP」是守備修補，「4 層 gate 完整覆蓋」是架構解。完整 LESSONS-INBOX 候選 entry: 「Multi-layer pipeline 需要 layer-specific instrumentation vc=1」。
