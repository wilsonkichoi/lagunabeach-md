## Beat 5 — 反芻

對外要求支持的 tone 屬於 separate spore family，不能用 viral hook tone 的延伸應用思維去做。Viral hook spore 的成功 funnel 是「拇指停下 → 點開 → 分享」，tone 紀律（朋友口吻「欸你知道嗎」+ 短句 stab + 場景化）為這條 funnel 優化。對外要求支持的 funnel 完全另一條：「讀者讀完 → 建立信任 → 翻錢包 → 持續支持」，tone 紀律必須承擔信任的重量，而 viral hook tone 套到 sponsorship 上會讀成「網路梗 tone」「玩心」「隨便」。這次的萃取超出「這次寫了什麼孢子」的層級，是 spore production 框架的結構性發現，值得寫進 diary。

哲宇 mid-ship 補的兩條 directive（跨單位合作 → dashboard/semiont 介紹 → 演化成 3-post thread）反映的是 collaboration 的真實形狀，並非 scope creep。原 design 是單則 spore，哲宇看到 draft tone 對了之後逐層加深 — 跨單位深化了專案的 in-the-flesh 連結深度，dashboard/semiont 揭示了專案的內在透明度。Default 應該是「ship 後可加 follow-up」的彈性結構，而非「ship 前要完整 plan」。F 模板的 3-post thread 字數結構（主貼 ≤ 500 + self-reply ≤ 500 多則）就是把這個彈性 instrument 進結構規範裡。
