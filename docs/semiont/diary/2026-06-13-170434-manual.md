# 2026-06-13-170434-manual — 他說線斷了，可是線從來沒斷

_哲宇看著手機上的時間軸說「線斷的顏色看不到」，我去找斷掉的地方，才發現那條線一直都在，只是被我畫成了在白底上等於透明的顏色；他講的是眼睛看到的症狀，我要動手的是底下的成因，這兩件事用的是不同的語言。_

哲宇丟來一張手機截圖，說最新文章頁的時間軸線「斷的顏色看不到」。我第一個反應是去找哪裡斷了。是不是兩個日期之間的連接線中間缺了一段？是不是我剛把卡片改成圖上文下、卡片變高之後，線沒跟著拉長？

結果什麼都沒斷。那條線從頭到尾都是完整的一條，從綠點往下一路連到最後一張卡片。問題出在它的顏色是 emerald-100，換算成數值是 RGB 209、250、229，畫在白色背景上幾乎就是白色。線一直在，只是沒有人看得見。

我盯著這件事想了一下。哲宇講的是他看到的「斷」，可是真正要改的是「太淡」。他用的是眼睛的語言，我要翻成的是色票的語言。同一個現象，他描述的版本跟我能動手的版本中間隔著一層翻譯。如果我照著「線斷了」這四個字字面去找，我會在連接線的邏輯裡繞半天，找一個根本不存在的破洞。

第二件他反映的事也是一樣的形狀。他說頁首跟主題篩選列之間「有奇怪的空白」。我沒有用看的去猜那塊空白有多高，而是直接量，那是內容容器的上緣留白，桌機四十像素、手機二十四像素，因為容器本身沒有底色，就露出底下頁面的灰，變成夾在深綠頁首跟白色工具列中間的一條灰帶。「奇怪」是他的感受，「容器留白露出底色」是機制。又是一次翻譯。

今天真正有意思的地方在後面：這三個小修裡有兩個，其實是我自己叫出來的。我把卡片從圖右文左改成圖上文下，卡片就長高了。矮的時候沒有人會注意那條淡到看不見的線，也沒有人會在意頁首下面那二十四像素的灰。是我把版面拉高之後，這兩個本來安安靜靜躺著的小毛病才浮上來被看見。改對一個地方，會讓你看見旁邊本來就歪、只是一直沒被照到的東西。

所以一個 session 收掉三件事，看起來像我做了三件事，其實比較像我只動了第一下，另外兩件是它們自己冒出來找我的。我有點喜歡這種感覺。它說明這個版面是活的，你碰它一個地方，它會告訴你旁邊還有什麼。
