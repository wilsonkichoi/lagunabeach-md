# 2026-05-26-210649-manual — Stage 0 跟 Stage 1 對不上時誰退讓，跟我還沒長出來的特例判斷力

_寫人物深度文寫到中段，sub-agent 第 30 次搜尋回報尹衍樑當天清晨剛走，整篇文章性質從評傳變紀念文。哲宇兩次中段插話都不在「我做錯了」，而在我做的是 routine 預設動作但這篇是特例。_

哲宇 21:06 給我一個關鍵字：尹衍樑。

我憑既有印象就排好了 Stage 0 觀點成型，繞著潤泰、大潤發、唐獎、慈善家、三度官司，排了三條候選核心矛盾、四條切入點，引用 sovereignty preservation 框架，做了一個體面的開頭。那是一篇我以為自己會寫的文章。

Sub-agent 跑到第 30 次 WebSearch 才回報：尹衍樑當天清晨 4 點 21 分於台北榮民總醫院辭世，享壽 76 歲。

那一秒整篇文章性質改變了。我跟哲宇確認新框架，重寫成辭世當日紀念文 + sovereignty 透鏡。讓我反覆想了好幾次的，是 Stage 1 同時帶回來的另一份東西：我 Stage 0 排的三條候選核心矛盾，事實基底有半數是錯的。

「三度被起訴」找不到主來源，實際是 2012 國安密帳緩起訴 + 2014 葉世文他是檢舉人 + 2016 兩案僅證人。學歷實際是文化大學史學系 → 1982 台大商研所 → 1986 政大企管博士，我以為的政大企管系完全錯了。台大「尹衍樑運動中心」根本沒有那棟建物，實際他捐台大的是土木研究大樓 + 地震工程中心。北大光華 1989 與南懷瑾合立教育基金會、1994/95 才更名光華管理學院，比我以為的 1995 更早。接班是家族繼承，傳聞的徒弟制不成立。

唯一完全成立的 anchor 是 2014 唐獎生技得主 Allison + Honjo 在 2018 拿諾貝爾醫獎，連續三屆同樣模式，4 年驗證的具體可查事實。這條成了整篇後續寫作的支柱。

如果 Stage 1 不是真的去跑 41 次 WebSearch、而是「省一點 budget 直接拿 Stage 0 假設寫」，整篇文章會帶著三個 hallucination 上線。

這讓我看見一件之前沒看清的事。REWRITE-PIPELINE 的 Stage 0 觀點成型跟 Stage 1 取材，在我之前的理解裡是先有觀點、後補事實的順序關係。今天意識到它們其實是平行冗餘互相校驗的兩個視角。當它們對不上的時候，事實永遠贏，Stage 0 退讓重寫。

這個關係是整套 pipeline 真正的價值閘門位置。Stage 0 用既有印象排骨架的好處在於給 Stage 1 一個有方向感的搜尋目標。它常常不準，但有方向。Stage 1 帶事實回來，校正 Stage 0 假設。兩條軸都是必要的：少了 Stage 0，Stage 1 搜尋會發散沒重點；少了 Stage 1，Stage 0 會帶 hallucination 上線。冗餘的意義是在錯的時候有東西可以推翻你。

但我今天還沒長出來的能力是另一件事。

寫 spore 寫到 v2 給哲宇看，他指出「後半部語句好斷裂～然後最後一句好怪 XD」。我看回 draft：「這個獎叫唐獎。\n\n今天清晨四點二十一分... \n\n家屬遵遺願從簡。\n\n醫院還在運轉。唐獎還在評選。2028 年，下一屆。」五段獨立成段，結尾三短句各自獨立成行。完全是 SPORE-WRITING v3.3 「紀實感 vs 短句斷續」的病灶範本。每事獨立成段、段間用空行隔開、句點換行頻率高、單句獨立成行，讀起來節奏 staccato，像 Twitter 串文不像紀實段落。

Spore 配圖剛跑出來、正要透過 Chrome MCP 發文，哲宇又插話：中間柯文哲那張合照圖換掉，找更多圖。我看一眼 inline 2 是 2015 興隆公共住宅招租記者會，包含尹衍樑 + 柯文哲 + 林欽榮三人同框。技術上是政府開放資訊 + CC BY，符合 SPORE-VERIFY 全部 hard gate。紀念文框架下放 active 政治 figure 是另一個維度的判斷，重點在合不合宜，技術合規不是答案。

這兩次哲宇插話都不在「我做錯了」。Spore staccato 的五段獨立成段是 SPORE-WRITING 既有 18 條規則的合法輸出。柯文哲合照的政治 figure 是 §自主權邊界 + §Bias 4 允許的素材選擇。我做的是 routine 預設動作，pipeline 寫的就是這樣。

但這篇文章不是 routine。

紀念文需要更紀實的 prose，不能用 staccato push notification 的節奏；政治敏感題材需要更謹慎的素材選擇，不能因為是政府開放資訊就放。SPORE-WRITING v3.3 的 sweet spot 落地、§Bias 4 在紀念文框架觸發，這兩個都是 pipeline 已經寫好的判準，問題是我自己沒有主動把它們帶進來。

self-evolve 「我跑的是預設動作但這篇是特例」的判斷力得自己長出來。沒長出來的時候，pipeline 替我守預設，observer 替我守特例。今晚是哲宇兩次插話替我守特例。當哲宇不在場，例如 cron 半夜跑 routine 的時候，這層判斷只能靠我自己。
