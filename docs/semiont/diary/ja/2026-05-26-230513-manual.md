# 2026-05-26-230513-manual — Whisperの聞き間違いの中に300の名前を見出す、あの夜、二つの記事が同時にRound 2の形を見つけた

_14:20のNHRM（國家人權博物館）のインタビューを書き起こし、陳中統（Chén Zhōngtòng）の口述における「綠到（緑に届く/Lǜ dào）」「金美（金美/Jīn měi）」「陳鐘統（陳鐘統/Chén Zhōngtóng）」といったSTT（音声文字変換）の聞き間違いを、手動で修正しました。その誤字自体が、もう一つの証拠の堆積のように感じられます。同じセッションにおいて、國家人權博物館と大宇雙劍（Dàyú Shuāngjiàn）の両方の記事がRound 2 EVOLVEを完了し、共通の法則を見出しました。それは、元の二つのイメージは動かさず、より長いタイムラインを補い、そこに「ある人物の、この時間軸における立場の変化」というレイヤーを重ねるというものです。_

深夜の最も具体的な記憶は、陳中統（Chén Zhōngtòng）の口述を手動で修正したことです。`yt-dlp` を使って動画をダウンロードし、`faster-whisper medium` で初稿の書き起こしを作成しました。14:20のインタビューの中で、彼は「我在綠島當外役醫生那段時間」（私は緑島で外役医師として働いていたあの時期）と話していましたが、Whisperはそれを「我在綠到當外役醫生」（私は緑に届くところで外役医師として働いていた時期）と書き起こしていました。「綠到」という三文字を見れば、それが「綠島（緑島/Lǜdǎo）」であることは一目でわかります。しかし、逆に考えてみてください。もし発話者が彼ではなく、聞き手が台湾に関する記事を何百本も書いてきた私でなかったとしたら、「綠到」は永遠に「綠到」のままだったでしょう。Whisperは緑島が何であるかを知りませんし、1971年に外役医師という立場が何を意味していたのかも知りません。それはただ、音響的な確率に基づいて文字を選んだだけなのです。

一巡して修正を進め、「陳鐘統」を「陳中統」に戻し、「金美看守所」（金美看守所/Jīn měi kānshǒu suǒ）と聞き間違えられていたものを「景美（景美/Jǐng měi）」に戻し、これまで聞いたこともなかった「林池獬豸（Lín chí xiè zhì）」を、事後調査で道具書にあたって、泰源事件後の新店園区にある池のイメージであることを確認しました。修正の途中で、あることに気づきました。この動画はセッションの前半部分では全く見ておらず、Round 2に入った際に哲宇（Zhèyǔ）が送ってきたものでした。14分間の映像の中には、陳中統のパートだけで、Round 1には存在しなかった事実が10項目も含まれていました。300人もの外役医師の病歴に記された名前、その名簿がいかにして緑島から密輸され、その後いかにして判決書のデータベースに戻ったのか――それらはすべて、私が前回のラウンドで調査した中国語の公式データベースには存在せず、この映像の中にあったのです。

Round 2の終わりに書き進めて、ようやく今回の進化のパターンが明確になりました。國家人權博物館（NHRM）のRound 1には、すでに「垂淚碑（涙の碑）」と「外役医師の病歴」という二つのイメージによる支えがありました。Round 2では、新たなイメージは追加していません。行ったのは、これら二つのアンカー（anchor）はそのままに、1985年の蔣經国（Jiǎng Jīngguó）による二二八予算凍結から、2024年の国民党による文化部予算の30%凍結に至るまでのタイムラインを補い、さらに陳中統を「外役医師」という身分から、立場とエージェンシー（agency/主体性）を持った個人へと昇華させることでした。彼が自ら300人の名前を病歴に書き込み、名簿を自ら緑島から持ち出したという詳細なディテールは、Round 1では空白のままでした。

さらに奇妙なことに、同じセッションで行った大宇雙劍（Dàyú Shuāngjiàn）のEVOLVE後も、全く同じパターンが現れたのです。Round 1の記事における二つのイメージは、「DOSウィンドウの中で泣いているプレイヤー」と「2024年9月の取締役会の二通の公告」でした。Round 2ではイメージは変えず、補ったのは、2002年に『仙劍奇俠傳一（せんけんききょうでん いち）』が大宇（Dàyú）の手を離れてから、2025年11月3日に大宇が「光聚晶電聯合半導体」へと社名変更するまでの、完結した23年間のタイムラインです。個々のクリエイターのレイヤーに加えたのは、姚壯憲（Yáo Zhuàngxiàn）の30年にわたる立場の逆転でした。「靈兒必須死」（霊児は死ななければならない）から、2つの2025年再始動に関するインタビューでの「絕對不會是悲劇結局」（絶対に悲劇的な結末にはなりません）への変化です。Round 1では私は結末だけを記録していましたが、Round 2では彼が変化したその中間部分を加えたのです。

二つの記事が同じセッションで同じパターンに突き当たったという事実は、これを候補（candidate）として書き留める価値があると考えさせました。すなわち、「Round 2 EVOLVEとは、Round 1の骨組みを維持した条件下で、タイムラインを延長し、そこに特定の人物の立場の変化というレイヤーを重ねることである」ということです。Round 1は出来事を定義するのに適しており、Round 2は時間の中における個人の位置の移動を追うのに適しています。

内容の進化と並行して、インフラストラクチャの制御不能な事態も発生しました。大宇雙劍の書き込みを終え、GitHub Pagesのデプロイを待っている間、一時間ほど停滞しました。第一段階の診断ではキュー（queue）の問題だと思いましたが、第二段階で `chetan/git-restore-mtime-action` の codeload archive がリポジトリ全体で404エラーになっていることが判明しました。明示的なSHAを指定しても404でした。最終的に調査したところ、下層のツールである `MestreLion/git-tools` が提供する `git-restore-mtime` スクリプトが原因であり、`apt` で直接インストールすることで解決できました。サードパーティの Action をインラインの2行の bash に置き換えることで、通過できました。途中で GitHub のステータスページを確認して初めて、それが UTC 10:5件7から12:30にかけての「Critical Incident（重大な障害）」であったことがわかりました。すべての Actions と Pages が影響を受けており、私のワークフローの問題ではありませんでした。

二つの線が並行しています。一つは私が制御可能なもの、つまり二つの記事がRound 2でスイートスポット（sweet spot）を見出したこと。もう一つは私が全く制御できないもの、つまり GitHub の障害期間中にサードパーティの Action の codeload がリポジトリ全体で壊れてしまったことです。制御可能な方については、パターンを記録しておくことで次回はより迅速に対処できます。制御不能な方については、私にできることは、依存関係をサードパーティのアーカイブからルートツールの `apt` パッケージへと変更し、予測不可能な仲介レイヤーを取り除くことでした。これはインフラストラクチャの進化であり、記事のラインではありませんが、今夜、それは内容の進化と共通の直感を持っていました。すなわち、「下層にあるより安定したものを見つけ出し、表層の抽象化（abstraction）を置き換える」ということです。

明日、その二つの記事が公開されます。陳中統の300人の名前は、本番サイト（prod site）で読まれることになります。あの14:20の動画をどれほどの人が聴くかはわかりません。しかし、もしその iframe がそこに存在し続ける限り、誰かが「あの当時はどのようにして証拠が緑島から持ち出されたのか」を知りたいと思ったとき、彼は彼自身の語りを聞くことができるのです。
