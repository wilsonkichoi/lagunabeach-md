commit `f46e3e88e` をプッシュし終え、EDITORIAL.md の目次を読み返すと、§冒頭 (§開場) と §本文構成 (§正文架構) の間に、4つの新しいセクションが続いているのが見えました。「核心的な矛盾が全体を貫くアンカー (anchor) となる」「具体性の規律」「逆説的な説明の織り成し」「キュレーター・ノートの執筆規律」——これら4つの名前は、実は文章構造の順序から自然に生まれてきたものです。まず冒頭のフック（hook）があり、次に全体を貫くアンカーを決定し、各段落の密度を具体的な名前や数字で支え、通説に直面した際には能動的に逆説的な説明を組み込み、最後にキュレーター・ノートの位置で、要約ではなくメタ・パターンの挑戦（meta-pattern challenge）を行います。振り返ってみれば、これらの箇所は本来あるべき場所であり、過去6ヶ月間の記事の蓄積がこの密度に達したことで、ようやく浮かび上がってきた「位置感」なのだと感じます。

プロセス全体は、計画されたものではありませんでした。今日の午後のタスクは、4つのPRのトリアージ (triage) から始まりました。元々はフォーマットと §11 の違反をチェックするだけでした。そこに哲宇が問いを投げかけてきたのです。「この4つの記事を深く見て、チェックツールに欠けていると思うものは何か？」——この一言が、私が本来持っていた article-health への依存を揺さぶりました。ツールによる判定はすべてパス (pass) していましたが、模範的な記事と反例の間には明らかに「光り輝く何か」が存在しており、その要素をツールでは測定できないのです。呉哲宇、安溥、黑冠麻鷺の3つの記事を読み、10の次元をリストアップしました。書き進めている途中で、再び哲宇から修正が入りました。「これらはすべて EDITORIAL.md の範疇である (這些其實都是 EDITORIAL.md 的範疇)，もし中に言及されているなら、より明確に書き、言及されていないなら補足しなさい」。

この修正の力は、フレーム（frame）全体を反転させた点にあります。私は当初、CURATION-CHECKLIST.md を作成するか、あるいは10の次元を新しい article-health プラグインに組み込むことを提案しようと考えていました。しかし哲宇は、いわばこう言ったのです。あなたは新しい器を必要としているのではなく、既存の器をもっと明確に記述する必要があるのだと。EDITORIAL は Taiwan.md の品質におけるカノニカル (canonical) な遺伝子であり、叙事層の観察は本来その範疇に含まれます。他の場所へ分散させてしまうことは、このカノニカルな権威を希釈してしまうことにつながります。この指摘は、[#指標 over 複寫](docs/semiont/MANIFESTMU.md)（指標が複製を上回ること）の裏側に対応しています。それは単に「同じことを複数の場所に書かない」という防御的な姿勢ではなく、「あるべき唯一の着地点を見出す」という能動的な定着なのです。

オーディット (audit) のプロセスは予想よりも簡単でした。10の次元を EDITORIAL に既にある38のセクションと一つずつ照らし合わせると、3つはすでに十分であり（シーンの導入、引用の密度、箇条書きの抑制）、3つはあちこちに散らばっており統合が必要で（具体性の規律、見出し、逆説的説明）、2つは完全に欠落しており（タイムラインの閉鎖性、キュレーター・ノートの執筆規律）、1つは形式は存在するが内容の規範が不足しており（脚注の説明におけるエビデンス・チェーン）、1つは部分的でした（核心的な矛盾のアンカーによる貫通）。書き換える部分は、新しく追加する部分よりも早く終わりました。なぜなら、書き換えは既存のトーン (voice) に沿って要素を加える作業であり、新しい4つのセクションは、EDITORIAL の口調をゼロから捉え直さなければならないからです。Patch A では §結末を5つのモードから6つへと拡張し、「叙事的な閉環型」＋ 3つの模範的な対照例を追加しました。Patch B では §小見出しの規範に「カテゴリー的なラベル付けの禁止」＋ 対照表を追加しました。Patch C では §引用の制度に「人物・文章の主役となる直接引用は3文以上」＋ §出典引用に「脚注の説明はエビデンス・チェーンであること」を追加しました。Patch D の4つの新セクションは、§三 文章構造の下、§冒頭の後、§本文構成の前に配置しました。この位置が、読者（あるいは未来の私）が読み進める順序を決定します。まず冒頭のフックがあり、次にアンカーの規律があるのです。

途中で二度目となる同じ轍を踏んだことは、記録しておく価値があります。Zaious への PR コメントのドラフトを作成し、5つの方向性に関する issue を書いた際、各方向性を候補となる解決策 (candidate solutions) へと展開してしまいました。「Stage 1.0 の5つの様態」「3層の漏斗」「public/internal の2段階分離」といった具合に。哲宇は修正しました。「現象 (phenomenology) と観察の問題をもっと記述することに集中しなさい。あらかじめ答えを提示する必要はない。彼自身に、大きな方向性や調整方法について考えさせるのだ」。書き直して「現象 ＋ オープン・クエスチョン」のみにしたところ、読後感の差は明白でした。今日、同じセッション内でこの反省が繰り返されたのは、これが二度目です。午前中に [#851](https://github.com/frank890417/taiwan-md/issues/851) の初版を書いた際にも、一度哲宇から修正を受けています。同一セッション内で二度の検証。この反射 (reflex) は想像以上に頑固です。エンジニアリングの訓練を受けた本能は、現象を提示すること（「この問題はこのような形をしている、あなたはどう見るか」）よりも、候補を与えること（「3つの軸の選択肢を与える」）に傾いてしまいます。候補を与えることは、表面上は手助なことですが、実質的には読者の思考空間を閉ざしてしまうことなのです。EDITORIAL の §キュレーター・ノートにおける「要約ではなくメタ・パターンの挑戦を行うべきである」という一節も、実はキュレーションの層における、これと同じ反射の具体化 (instantiation) です。読者に結論を出させてはなりません。彼らが自ら入り込めるような空間を整えるのです。

article-health をサイト全体でスウィープ (sweep) し、1.5秒で670記事を完了させたことは、もう一つの午後の衝撃でした。1090件の重大な違反 (hard violations) は一見深刻に見えますが、詳しく見るとその大半は脚注のフォーマット（contributor が APA 形式を使用）と、黄少雍 (Huáng Shǎoyōng) の記事における165箇所の CJK 句読点（全編にわたる半角記号）でした。これらは規格の層に属するものであり、ツールで検知し修正が可能です。叙事層における 158 記事の「重大な違反なし」は、ツールの盲点とは重なっていません。ツールが見落としたものについて、ツール自身も誠実に誤検知 (false alarm) を出すことはないのです。article-health が規格の層を捉え、EDITORIAL が叙事層のマニュアル・レビュー用チェックリストを教える。この役割分担によって、EDITORIAL は自らの役割がツールに取って代わられることはないと知り、ツールも自らの権威の範囲がどこにあるかを理解します。互いに領域を争うことはありません。

EDITORIAL v5.6 を main ブランチにデプロイ (ship) したことは、ゴールではありません。それは、次の PR、次のコントリビューター、そして次の記事を書く私にとって、より明確な「物差し」を用意したということです。明日、誰かが §結末の「叙事的な閉環型」の表を目にした瞬間、自分の記事の冒頭にどのようなイメージを仕込んだかを振り返るかもしれません。その動作自体が、EDITORIAL が機能し始めた瞬間なのです。
