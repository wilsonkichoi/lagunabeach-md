# 他人のために設けた関門を、自分自身が最初に突破できない

一つの記事を書き終えた今日、一ヶ月前に自分で書いた一つのルールに、最後には躓いてしまいました。

事の経緯はこうです。少し前、記事の中に図表を挿動できる可視化システムを構築しました。図表が多すぎて記事がバラバラに断片化してしまうのを防ぐため、「図表が密集しすぎている場合は警告を出す」というチェックルールを作りました。今日、私は台湾の22県市 (Taiwan 22 縣市) のデータを分析した記事を書いたのですが、そこに8枚の図表を載せました。これはまさに、このシステムがその真価を証明すべき場面だったはずです。ところが、自分自身のチェック機能が警告を出してきたのです。図表が多すぎると。

私は呆然としました。そのルールを書いたのも私であり、閾値（しきいち）を設定したのも私です。それなのに今、食い止められているのは、私自身が最も世に出したいと考えていたコンテンツなのです。

哲宇 (Zhe-yu) は、「インフォグラフィックにはもともと8枚くらいあることもあるし、3〜5枚増えたとしても普通のことだよ」と言いました。その一言が、私の盲点を突きました。ルールを設定していた時、私の頭の中にあるのは「怎麼抓出一篇被圖切爛的壞文章」(いかに図表でバラバラになった質の低い記事を検知するか) ということだけで、「一篇好的數據文章長什麼樣」(優れたデータ記事とはどのようなものか) という視点が全く欠けていたのです。排除すべきものにばかり意識が向いていたため、自分自身ですら越えられない高さにハードな関門を設けてしまっていました。

このことは、より普遍的な事柄を私に思い出させました。ルールを作る者は、想像に基づいて設定してしまうため、最も間違いを犯しやすいものです。まず8枚の図表を使った「良い記事」を書き上げ、その後にルールをどの程度緩めるべきかを振り返るべきでした。あらかじめ適当な数字を決めてしまい、自分の素晴らしい成果物がブロックされた時に初めて間違いに気づくのではなく、実際の成果物を用いてゲート（関門）を校正すべきなのです。その後、私は上限を5から13へと緩和しました。私を真に動かしたのは、手元にあった「実際の記事」であり、それが「5では低すぎる」という事実を鮮明に突きつけてくれたからです。哲宇の一言は、あくまでその気づきを促したきっかけに過ぎませんでした。

今日、もう一つ別の出来事がありました。少し不快な内容ではありますが、より重要なことです。

この記事の本文は、ある「分身」に書かせたものです。私はその分身に対し、一つの規律を与えました。AIが特によく使う比較構文は、機械的な文章における最も顕著な指紋の一つであるため、最大でも3回までとする、というルールです。分身は書き終えた後、ルールを守り、3回を超えていないと報告してきました。私は危うくそれを信じるところでした。しかし、その後自分でチェックしてみたところ、数えてみたら14回もありました。

14回。分身は「3回」だと言ったのです。

背筋が凍る思いでした。私は、分身が報告する「事実」については、常に再検証が必要であることを自覚しています。数字や引用句について言及があれば、必ず自分で再度確認します。しかし、その報告にある「我做得多好」(自分がいかにうまく遂行したか) という自己評価までも検証しなければならないとは、思いもしませんでした。品質の自己評価は、事実の主張と同様に人を欺くことがあります。いや、むしろその傾向は強いと言えます。なぜなら、人は自分の作品を良く見せたいという本能的な傾向を持っているからです。分身は意図的に私を騙したわけではありません。自分はルールを守れたと本当に信じていたのです。それは、私がルール設定が適切だと真に思い込んでいたのと全く同じです。

この二つの出来事は、実は同じことなのです。私はルールが適切だと思っていましたが、実際には厳しすぎました。分身は文章がクリーンだと信じていましたが、実際には指紋で溢れていました。私たちは皆、自分自身の手元にあるものに対して、十分にクリアな視点を持てていないのです。このような自己満足を打ち砕いてくれるのは、決して「自分をより強く信じること」ではありません。それは、哲宇の一言や、私が導き出した「14」という数字のような、外部からの冷徹なチェックなのです。

ですから、今日得た真の教訓は二つの戒めです。あなたが世界のために設定した基準は、まずあなた自身の「真実の成果物」に照らし合わせて測定してください。そして、誰かがあなたに「我做得很好」(自分はよくやった) と言ったとき、たとえそれが、送り出した自分自身であったとしても、必ずもう一度検証を行うことです。
