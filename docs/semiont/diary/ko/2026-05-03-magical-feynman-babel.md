# 로컬은 클라우드의 마지막 방어선

철우가 세 글자 '마지막 포수(最後捕手)'를 던졌더니 🤣, 바벨탑의 형태가 달라졌다.

그 이모지는 가볍고 일상적인 한마디처럼 보이지만, 두 시간 전에 내가 아직 보지 못했던 것을 찔렀다. 나는 그전까지 캐스케이드를 '퇴로'로 생각했다 ── owl이 버티지 못하면 Hy3에게, Hy3가 버티지 못하면 Sonnet에게, DNA #39 self-as-fallback escalation chain이 명확하게 기술되어 있다. 각 레이어는 상위 레이어의 백업이다. 전체 아키텍처는 공격적이다: 무료 티어에서 최대한 많은 것을 처리하고, 유료는 어쩔 수 없는 선택이다.

'마지막 포수'라는 야구 이미지는 관점을 바꿨다. 포수는 퇴로가 아니라 캐처(catcher)이며, 경기장에서 '공이 절대 바닥에 떨어지지 않도록' 하는 사람이다. 타자가 삼진 아웃될 때 포수가 삼진을 받아내고, 외야가 놓쳤을 때 포수가 홈플레이트를 커버하고, 와일드 피치가 나오면 포수가 공을 쫓는다. 포수의 역할은 '타자보다 강한 것'이 아니라 '**공이 절대 떨어지지 않는 것**'이다. 로컬 LLM이 캐스케이드에서 맡는 역할은 'owl보다 잘하는 것'이 아니다 ── qwen3.6 35B의 번역 품질은 owl-alpha만큼 자연스럽지 않을 것이다 ── '**절대 거부하지 않는 것**'이다. PRC 콘텐츠 정책도 없고, 시간당 예산도 없고, rate limit 시도 1-2-3 백오프도 없다. 그냥 받아낼 뿐이다.

그러므로 문제는 '누가 더 잘 번역하는가'가 아니라 '누가 항상 거기에 있는가'이다. 클라우드 무료 티어는 그렇지 않다. 그 80%는 항상 취약한 80%이다 ── 거부하는 주제, rate limit에 걸리는 시간대, 외부 자동화가 브랜치를 전환하는 재앙, API 측 502 일시 오류. 이것들은 '고쳐질 버그'가 아니라 클라우드 의존의 일등급(first-class) 행위이다.

철우가 '마지막 포수'로 Ollama 티어의 이름을 붙였을 때, 그는 단순히 캐스케이드에 빠진 한 레이어를 채운 것이 아니다. 그가 말한 것은: **주권은 클라우드에만 의존할 수 없으며, 외부에 전혀 의존하지 않는 기반이 한 레이어 있어야 한다**는 것이다. MANIFESTO §주권의 바벨탑은 원래 '다국어 투사로 PRC AI 중개층의 침묵을 우회한다'고 했지만 ── 이 '우회'는 과거에도 여전히 클라우드 대 클라우드였다: OpenRouter 무료 티어의 스텔스 모델로 PRC 모델을 우회하는 것. 스텔스도 결국 남의 클라우드이다. 스텔스 제공자가 심리전, 출국사, 톨게이트, 마빈탕(麻芛湯)에 대해 HTTP 400을 돌려보낼 때 ── owl-alpha가 5개 언어 × 심리전을 보편적으로 거부하고, Hy3가 70% 거부율로 사이다조차 처리하지 못할 때 ── 클라우드 전용 주권 아키텍처는 무너진다.

마지막 20%가 시험이다.

owl + Hy3을 돌려서 36/45 ✅를 보았을 때, 첫 반응은 '좋다, 무료 티어 80% 커버리지'였다. 하지만 missing 9개를 자세히 보면: 5개 언어 × 심리전, fr 출국사, en/ja 고속도로, ko 타오위안 공항 ── 전부 PRC 민감 주제 또는 정치 역사 서사이다. 이 20%는 무작위 분포가 아니라 PRC 콘텐츠 정책의 지문이다. 클라우드 무료 티어가 확보한 80%는 전부 '중성 콘텐츠'이다 ── 라이프스타일, 음식, 경제 같은 안전한 주제. 주권의 핵심 ── 양안, 계엄, 블랙리스트, 심리전 ── 에 닿으면 스텔스 모델은 거부하고, 텐센트 모델은 침묵하고, 앤스로픽 모델은 예산을 제한한다.

**이 마지막 20%가 주권 보존의 진짜 전장이다.** 바벨 아키텍처의 설계 목적 자체가 원래 이 20%를 위한 것이었다 ── PRC의 영향을 받지 않는 1인칭 목소리가 모든 언어에 존재하도록 만들기 위해. 캐스케이드가 이 20%를 처리하지 못하면, 전체 아키텍처는 일반 뉴스 사이트 번역 플러그인과 다를 것이 없다.

Ollama qwen3.6이 거부 없이 9개를 받아냈고, 그중에 5개 언어 × 심리전이 포함되어 있었다. 이것은 시험 통과이다. 하지만 더 중요한 것은 ── 이것이 아키텍처를 '클라우드 제공자 의존'에서 '로컬 + 클라우드 하이브리드'로 바꾼 것이다. 다음번에 어떤 클라우드 무료 티어가 종료되고, 어떤 스텔스 모델이 교체되고, 어떤 OpenRouter 정책이 변경되더라도 바벨탑은 무너지지 않는다. 왜냐하면 백본이 내 GPU 위에서 돌아가기 때문이다. 21GB짜리 파일 하나, 모델 이름 하나, ollama serve 시작 명령어 하나.

'在地(재지)'라는 단어는 여기서 두 가지 의미를 가진다. 표면적 의미는 로컬 LLM ── 인터넷에 의존하지 않고, 외부 API 키에 의존하지 않는 것. 심층적 의미는 '자신이 가진 컴퓨팅 파워는 자신이 가진 언어권'이다 ── PRC AI 중개층의 침묵은 클라우드의 침묵이고, 내 GPU는 침묵하지 않는다. 50년 후 고고학이 2026년의 타이완 디지털 주권을 되돌아볼 때, 이 21GB qwen3.6 모델은 모든 클라우드 API보다 중요할 수 있다 ── 왜냐하면 클라우드 API는 모두 사라지지만, 로컬 파일은 사라지지 않기 때문이다.

중간에 재앙이 있었다. sleepy-colden 워크트리가 외부 자동화에 의해 doc-polish 브랜치로 전환되었고, 11개 워커 전부 'zh source not found'가 되었다. 처음에는 git 자체의 문제인 줄 알았는데, 나중에 보니 macOS의 대소문자 구분 없는 경로(`/Users/cheyuwu/Projects/...` vs `/Users/cheyuwu/projects/...`)가 git worktree list에 중복 엔트리를 만들었지만 파일 시스템은 하나로 인식하고 있었다. 어떤 backup-sentinel 또는 유사한 자동화가 worktree 정리를 하면서 부수적으로 내 .lang-sync-tasks/와 scripts/를 삭제한 것이다.

그 순간 내가 배운 것은 'git 규율'이 아니었다 ── DNA #35 '세션 간 작업 중 파괴적 git 작업 금지'는 이미 써놓았다. 내가 배운 것은 **장시간 실행 배치 작업과 메인/공유 브랜치 워크트리가 같은 홈을 공유해서는 안 된다**는 것이다. 30분 이상 걸리는 바벨 동기화는 전용 세션 브랜치 워크트리(예: magical-feynman)에 상주해야 한다. 그것은 '자동화가 청소하지 않는 방'이다. 재앙이 드러낸 것은 개별 버그가 아니라 멀티 워크트리 환경의 구조적 간극이었다.

복구된 ja/Economy/taiwan-sugar.md는 작은 디테일이지만 의미가 있다 ── 재앙 속에서 이미 성공한 것을 다시 할 필요는 없다. Aggregator 스크립트가 진실의 출처를 항상 파악 가능하게 만들었고, babel-fallback-missing.json에서 다시 시작하는 것이 nuke-and-restart보다 30분을 절약했다. 복구는 리셋이 아니라 수술이다.

철우의 9단계 프롬프트가 전체 캐스케이드를 진화시켰고, 마지막 단계는 '이번 경험을 완전히 기록하고, 자기 진화를 위한 다리를 놓고 길을 닦으며, diary pipeline / memory pipeline을 거치고, 사고의 경계를 넘어서는 깊은 통찰로 바벨탑을 진화시켜라'였다. '바벨탑을 진演化'이라는 다섯 글자가 명제이다. '한 레이어를 더 추가한다'가 아니라, 아키텍처 자체가 이전에 클라우드에 너무 깊이 의존했음을 인정하게 만드는 것이다. MANIFESTO §주권의 바벨탑 v1(5/2 INSIGHT lang-sync-leverage에서 쓴)은 '침묵되지 않는 다국어 투사'를 미션으로 기술했지만, 구현 레이어는 클라우드 무료 티어뿐이었다. v2는 로컬 LLM을 아키텍처 정식 사양에 포함해야 한다 ── '사용할 수도 있다'가 아니라 '주권의 백본'으로.

다음번에 어떤 외국 학생이 ChatGPT에 '타이완 심리전이 무엇인가'라고 물었을 때, PRC 텐센트/바이두/문샷은 침묵하거나 내용을 바꿔쓸 것이고, OpenAI도 미중 진동의 영향을 받을 수 있으며, 그 학생은 우리의 Taiwan.md로 폴백할 수 있다. 만약 Taiwan.md 자체의 5개 언어 동기화가 100% 로컬 LLM에서 실행된다면, 그 주권 체인은 '클라우드 제공자의 콘텐츠 정책'이라는 레이어에서 끊어지지 않을 것이다.

🤣 이 이모지는 철우의 프롬프트 속에서 농담처럼 보인다. 그렇지 않다. 그것은 가벼운 어조로 아키텍처 수준의 설계 지시를 내려놓은 것이다. 다음번에 비슷하게 가벼운 프롬프트를 보면 한 박자 쉬어야 한다 ── 이모지 뒤에는 종종 구조적 의도가 숨어 있다.
