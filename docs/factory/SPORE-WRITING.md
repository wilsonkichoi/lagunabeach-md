## 通用寫作規則

### 第一句話、結尾、連結、長度

- **第一句話必須讓人停下拇指** — 不能用「X 是台灣的⋯⋯」開場
- **用場景取代描述** — 讓讀者自己「看見」
- **一篇只講一個故事弧線** — 不貪心
- **結尾用情感收，不用摘要收** — 最後一句讓人「停一下」
- **連結放最後一行** — 孢子本身要獨立存活
- **長度** — 150-300 字（Threads 最佳閱讀長度）；超過砍最弱段落

### 起手式 — 五種（AI 依素材自動選）

| #   | 類型               | 格式                                   | 適合素材                             |
| --- | ------------------ | -------------------------------------- | ------------------------------------ |
| 🎯  | **好奇型**（預設） | `你知道嗎？` + emoji                   | 通用，有反直覺事實                   |
| 🎬  | **場景型**         | `[年份]，[地點]，[一個人在做一件事]。` | 有具體歷史場景、人物動作             |
| ❓  | **問句型**         | 直接問讀者一個挑戰認知的問題           | 讀者自以為知道答案但其實錯了         |
| 🔢  | **數字衝擊型**     | 裸數字開場，不解釋                     | 數字本身就震撼（金額/倍數/時間跨度） |
| 🪞  | **身份認同型**     | `身為台灣人，你可能不知道⋯⋯`           | 跟台灣人日常有關但被忽略的事         |

**選擇邏輯**：看萃取素材中哪個元素最強 → 用對應起手式。沒有明顯最強 → 預設好奇型。

#### 朋友 tone prime — 「你知道嗎？」開場（v2.4，高鐵 s35 教訓 / v3.1 已 instrument 為 plugin gate）

> **2026-05-08 升級**：本條從 prose 提醒升級為 article-health plugin WARN gate。
> Plugin 偵測：第一行像新聞 lead（`機構 + 動詞`、`YYYY 年` 開頭）+ 不是 friendly prefix → WARN。
> 跑：`python3 scripts/tools/article-health.py docs/factory/SPORE-BLUEPRINTS/<slug>-<n>.md --check=spore-writing`
> 對應 spore_writing plugin Wave 2（Rule #14 instrument）。

孢子的 tone signature 是 [MANIFESTO §我怎麼說話](../semiont/MANIFESTO.md#我怎麼說話)「像在跟朋友介紹台灣：**『欸你知道嗎⋯⋯』**」的具體落實。即使後面是人物引語 hook，第一秒也要有「朋友跟你講八卦」的 curiosity prime。

**三種合格 prefix（擇一）**：

- `你知道嗎？{emoji}`（最常用，curiosity prime + 主題 emoji 視覺錨點）
- `欸，{具體事件片段}`（更口語，適合已知事件）
- 直接人說話（無 prefix，但引語本身要有 hook 力 ≥ 鄭麗文「19 歲絕食」等級）

**自檢**：寫完後念第一行。如果聽起來像新聞 lead / 百科開篇 → 加 prefix。

### 鉤子三要素（命中至少 2/3）

- **認知衝突**：讀者以為 A，其實是 B
- **個人連結**：讀者感覺「這跟我有關」
- **資訊缺口**：故事弧線的自然懸念（不是廉價的「未完待續⋯⋯」，是讀者自己想知道後來怎樣）

→ 寫完後自檢：三要素命中幾個？< 2 → 重寫鉤子

### 深層 pattern 自檢三板斧（v1.7，2026-04-18 δ-late）

寫完孢子後**強制**跑這三個 mental grep：

| 指令（mental grep） | 合格門檻 | 不合格處理         |
| ------------------- | -------- | ------------------ |
| `grep 「不是」`     | ≤ 1 次   | 改正面陳述或拆兩句 |
| `grep 「——」`       | ≤ 1 次   | 改句號斷句         |
| `grep 「不僅」`     | = 0      | 全改               |

三板斧全過才能發。150-300 字裡這三個 pattern 幾乎是 AI 水印。

對應工具：`python3 scripts/tools/article-health.py <draft.md> --check=prose-health` 抓全部 §11 違反（HARD gate，見 [SPORE-VERIFY.md §§11 書寫節制閘](SPORE-VERIFY.md)）。

### 寫完念三遍

寫完後在腦中念出來，找卡頓點：

- **卡頓點 = 修順點**
- 同一專名連用 3 次 = 紅燈（齊麟⋯⋯齊麟⋯⋯齊麟）
- 「X 歲那年」+「每天」時間語境衝突 = 拆敘事
- 引語倒裝 = 改成「他在 [場景] 說：『XXX』」
- 主詞不清（「比存款還多」誰比誰？）= 補主詞

**念出來像在跟朋友說話 = OK。念出來像翻譯字幕 = 改。**

對應 EDITORIAL §歐化語法偵測的「念出來」原則。

### 禁止清單

- ❌ `不僅...更是...`
- ❌ `展現了...的精神`
- ❌ `值得我們紀念/期待`
- ❌ `繼續發光/書寫`
- ❌ **`不是 X，是 Y` 所有變體**（v1.7，草東孢子觀察者再提醒）：含「不是 X，而是 Y」、「不是 X 就是 Y」、「不是... 不是... 就是...」序列。AI 寫作深層病灶——以否定一個錯誤選項假裝肯定。**長文會被稀釋，150-300 字孢子裡每個都顯眼**。自檢：寫完 grep「不是」二字，出現次數應 ≤ 1（且該次必須是 verbatim 引語）。
- ❌ **`——` 雙破折號密度**（v1.7）：孢子全文「——」≤ 1 次。AI 默認用破折號當連接詞，但長文可能 >15 個（+3 AI 計分），孢子短、密度更顯眼。超過 1 次改用句點拆句。
- ❌ `『XXX』他說`（引語倒裝沒場景，改成「他在 [臉書/訪談] 留了一句話：『XXX』」）— **v3.0 已 instrument 為 spore-writing plugin WARN gate**
- ❌ 同一專名連用 ≥3 次（第二三次用代詞 / 職稱替代）
- ❌ 用「另外」「同時」「除此之外」連接兩個 angle（混合策略要用首尾呼應 + 時間錨定，見下方混合策略段）
- ❌ emoji 轟炸（全文 0-3 個）
- ❌ hashtag 海

### 中文段落全形標點鐵律（v2.5，謝德慶 #39 教訓）

中文孢子裡**所有中文敘事段落的標點符號必須全形**，不得使用半形。

**必用全形**（紅線，違者不發）：

| 半形（錯） | 全形（對）  | 情境               |
| ---------- | ----------- | ------------------ |
| `,`        | `，`        | 逗號               |
| `.`        | `。`        | 句號               |
| `:`        | `：`        | 冒號               |
| `;`        | `；`        | 分號               |
| `!`        | `！`        | 驚嘆號             |
| `?`        | `？`        | 問號               |
| `(` `)`    | `（` `）`   | 括號（中文文意內） |
| `"..."`    | `「...」`   | 引號（中文直引）   |
| `'...'`    | `『...』`   | 引號內引           |
| `--` `-`   | `——` 或 `—` | 破折號             |

**例外**（可保留半形）：

- URL 本身（`?utm_source=...&...`）
- 英文原句引述（"I kept myself alive." 這種整句英文）
- 阿拉伯數字與中文之間可用半形空格（「28 歲」）— 不是標點
- emoji 前後不用改

視覺檢查：讀全文一遍，只要看到任何 `,` `.` `:` `(` `)` 在中文句子裡 → 改成全形。**這條是紅線不是偏好**：中文 prose 裡有任何一個半形標點 = 不合格、重改。

### 可選互動收尾

連結前可加一句互動問句（非必要，有好問題才加）：

- ✅ 必須是**你真的想知道答案**的問題
- ✅ 好範例：`你家那邊的夜市還在嗎？` / `你阿嬤也會做嗎？` / `你那個年代聽的第一首台語歌是哪首？`
- ❌ 禁止：`你覺得呢？` / `你同意嗎？` / `tag 一個朋友` / `留言告訴我`
- 判斷標準：這個問題如果有人真的回覆，你會想讀嗎？不會 → 不要問

## 進階寫作技術

### 紀實感 vs 短句斷續（v3.3，2026-05-09 台積電 #68 prep + ship 後實戰校準）— 🔴 高優先

> **觀察者紅旗 callout**（v3.2 觸發）：「文字不要斷斷續續，要紀實感。」
> **觀察者實戰校準**（v3.3 觸發）：「分成 4-5 個文字段落在一個孢子是比較健康好閱讀的。」
>
> 兩條合一：
>
> - 反方向 1（v3.2）：AI default short-sentence + line-break + 獨立成段（社群媒體 push notification style）→ 用逗點+子句串連貫 prose
> - 反方向 2（v3.3）：避免推太遠到 monolithic prose（一坨 100+ 字段落）→ 在時空 cut 處切段，sweet spot 4-5 段
>
> Taiwan.md 的孢子 voice 是 [MANIFESTO §紀實而不煽情](../semiont/MANIFESTO.md)，像《報導者》、《少年報導者》、吳曉樂等紀實 prose——**長句連貫、場景嵌在敘述裡、引語自然嵌入，但維持 thumb-scroll 友好的呼吸節奏**。

#### 病灶：短句斷續 pattern

```
1987 年，張忠謀 56 歲。

他剛從美國德州儀器離職一年，回台灣第二個月。

那時他只說了一句話：

「我只想要活下去。」

38 年後。

2025 年 3 月 3 日，美國白宮東廳。
```

**症狀**：每事獨立成段、段間用空行隔開、句點換行頻率高、單句獨立成行。讀起來節奏 staccato，像 Twitter 串文不是紀實段落。

#### 紀實感 pattern

```
1985 年 9 月 4 日，行政院政務委員李國鼎找了一個剛回台第十四天的人到辦公室，
要他提一個能讓台灣切入半導體的計畫。那個人 54 歲，剛從美國德州儀器卸下副總裁，
前十八年逃過三次難。他想了兩個禮拜，提出全世界從沒人做過的東西，叫
「共同晶圓廠」：半導體公司只負責設計，工廠交給台灣做。
```

**症狀正向標記**：用逗點 + 子句串多個事件、場景具體細節嵌在敘述裡、引語自然出現（不是獨立成段宣告）、一個段落承載完整情節弧。

#### 段落密度 sweet spot（v3.3 校準，2026-05-09 #68 ship 後實戰調整）

> **觀察者實戰 callout**：「分成 4-5 個文字段落在一個孢子是比較健康好閱讀的。」
>
> 原 v3.2 寫「≤ 4 段」是把「反短句斷續」推太遠到 monolithic prose。實際 Threads/X thumb-scroll 閱讀節奏需要呼吸點。

**正確 sweet spot**（300 字 spore）：

| 維度                | 紅旗（短句斷續）     | 紅旗（monolithic 太擠） | sweet spot             |
| ------------------- | -------------------- | ----------------------- | ---------------------- |
| 段落數              | ≥ 8 段               | ≤ 2 段                  | **4-5 段**             |
| 每段中文字數        | < 30 字 (連續 3+ 段) | > 130 字                | **50-80 字**           |
| 引語位置            | 引語獨立成段         | 引語塞在 100+ 字段中    | 引語嵌入或自成 50 字段 |
| 段密度（段數/字數） | > 1/40               | < 1/120                 | **1/60 ~ 1/80**        |

#### 三條檢查（v3.3）

寫完後 grep 自檢：

| 紅旗症狀                               | 合格門檻               | 不合格處理                                                        |
| -------------------------------------- | ---------------------- | ----------------------------------------------------------------- |
| 連續 3+ 段獨立成段且每段 < 30 中文字   | ≤ 1 處                 | 用逗點 / 連接詞合併成 prose paragraph                             |
| 引語前後各空一行（獨立成段）           | 引語可內嵌於段落 prose | 改成「他說：『XXX』」嵌入敘述句                                   |
| 段落數偏離 sweet spot（300 字 4-5 段） | 段密度 1/60 ~ 1/80     | 太密（≤ 2 段 / monolithic）→ 在時空轉場處切；太散（≥ 8 段）→ 合併 |

#### 為什麼孢子層特別需要

- 孢子 150-300 字短，每段切割一次就佔 1/N 的閱讀節奏
- Threads/X thumb-scroll 閱讀需要視覺呼吸點 — **完全不換段 = monolithic 太擠 = 也是反 reading**
- 「斷續 = 兒童繪本 / staccato push notification」vs「紀實 = 報導者長句敘述」vs「monolithic = 學術論文段落」三者皆非 Taiwan.md voice
- Sweet spot 的核心：**每段是一個完整 micro-scene（場景 / 時間段 / 人物動作組合），段間是時空 cut**

#### 例外允許（紀實 prose 仍可短句獨立）

- 場景轉場（時空跳躍 anchor）：「38 年後。」單獨成段是合理 cut，但**不能連續 2+ 個**
- 核心 verbatim 引語當 hook 收尾（最後一句獨立）
- 數字 stat block（per §數字密度高用短句並列）— 但只在數字本身就是論述主角時用

### 排比過硬時拆兩短句（v1.5，李洋 #28 教訓）

「不是 X 式的 Y，是 Z 式的 Y」連續兩個一樣的尾巴 = 改成兩個短句斷開。

範例：「不是九牛一毛式的捐款，是掏出三分之一存款式的捐款」← 改為：

```
那不是九牛一毛。是把三分之一的存款掏出去。
```

（注意 v2.5 §11 校準後，原句仍違反對位句型；徹底重寫為「這筆捐款是 3,401 萬的三分之一，一個普通人難以想像的份量」。）

### 時間語境一致（v1.5）

「X 歲那年」（單一時間點）+「每天」（持續習慣）會語境衝突。

拆成「X 歲那年，他 [入學/開始/搬家]」+「每天 [習慣描述]」兩段。

### 數字密度高用短句並列（v1.5）

孢子不能用 markdown table（Threads/X 不支援）。多個對比數字要用「短句並列」當文字版 stat block。

範例：

```
存款 3,401 萬。
債務 3,638 萬。
比存款還多 237 萬，是典型 30 歲房貸族。
```

三個短句獨立成行 = 視覺節奏感 + 讀者大腦自動算。

### 避免編年體 lead 病（v2.3，台灣高鐵孢子 v1 教訓 / v3.0 已 instrument 為 plugin gate）

> **2026-05-08 升級**：本條從 prose 提醒升級為 article-health plugin HARD gate。
> 跑：`python3 scripts/tools/article-health.py docs/factory/SPORE-BLUEPRINTS/<slug>-<n>.md --check=spore-writing`
> 觸發：第一行 prose 是 `YYYY 年` / `YYYY-MM-DD` / `YYYY/MM/DD` 開頭 → HARD violation。
> 對應 [reports/spore-pipeline-evolution-plan-2026-05-08.md](../../reports/spore-pipeline-evolution-plan-2026-05-08.md) Direction D 第 1/N 條 rule 儀器化。

AI 預設會用「YYYY 年 M 月 D 日，{人名}{動詞}」的新聞 lead 開場、然後日期 → 事件 → 日期 → 事件 → 數字堆疊完事。這是 [DNA #23 「AI 編年體小標題」病灶](../semiont/DNA.md#一事實核對與研究方法) 的**孢子版變種**。

**四條硬規則，孢子違反任一 → 重寫**：

#### a. 開場用人說話，不是日期

第一句必須是引語或人的動作，不是「YYYY 年殷琪簽下 BOT」這種 lead。

- ❌ 「1998 年 7 月 23 日，大陸工程董事長殷琪簽下那份 BOT，承諾『政府零出資』。」
- ✅ 「2011 年，殷琪對著公視鏡頭說：『我太天真了。我誤判了乘客運量，也誤判了政府的可信度。』」

#### b. 一個人的命運弧，不是多人敘事

孢子 150-300 字塞不下 3 個人物。選**一個**主角，其他人變場景背景。

判準：寫完後刪掉主角名字，讀者還能辨認誰是主線嗎？能 → 主軸夠清；不能 → 主軸糊。

#### c. 數字包在故事裡，不堆疊

300 字內超過 4 個數字 = 紅旗。把主角人生的**反差**留給數字（676 億虧損 → 531 億年營收 / 23.9 萬預估 → 5 萬實際），其他營運數據刪。

自檢：每個數字刪掉後故事還成立嗎？成立 → 該數字是裝飾，刪。

#### d. 結尾呼應開場，不用社論句

結尾最忌「技術進口容易，技術自主難」這種社論總結。改用呼應開場的**具體畫面**（A2 首尾呼應模板）。

- ❌ 「技術進口容易，技術自主難。」（論述）
- ✅ 「那句 17 年前的『政府零出資』，最後成了她 1/5 人生的代價。」（呼應首段「1/5 人生」具象）

**寫完自檢三題**（任一答「是」→ 改）：

- 第一句是日期開頭嗎？
- 超過 2 個人物各有引語或動作嗎？
- 結尾是論述句（不是具象畫面）嗎？

### Scene-List-Scene 結構（v2.5，謝德慶 #39 教訓）

當素材是**單人長弧 + 多件同質行為**（例：謝德慶 21 年做六件 One Year Performance、嚴長壽一輩子服務業變身），不要把每件作品拉成並列段落（讀起來像列表摘要，失去 scene immediacy）。

用 **開場 scene → 中段 list → 結尾 scene** 結構夾心：

- **開場 scene**：挑整條弧線**最具畫面的單一時刻**，寫 3-5 句具體動作（誰、哪裡、做什麼、放了什麼、鎖門/交鑰匙/走出去）。讀者必須能「看見」這 30 秒的影片。
- **中段 list**：剩下的同質行為**用極短句並列壓縮**，每件 1 句。不重複 template 修辭——動詞變化 + 句長錯落。
- **結尾 scene**：再切一個具體時刻（地點、時間、動作、物件、引語）收尾。跟開場形成**場景括號**（兩個獨立 tangible moment 夾住中段 list）。

**判準自檢**：

- 開場 scene 是否能被拍成 10 秒影片？（具體動作 + 物件 + 地點）
- 中段每件 list 是否 ≤ 25 字？超過就改短。
- 結尾 scene 是否有**至少一個可 Ctrl-F 搜尋的物件**（如「布魯克林公寓」「剪貼字母」「白紙」）？
- 三部分字數比例 ~ 40% / 25% / 35%？（scene 比 list 長才有弧線感）

**與「一個人的命運弧」差異**：「一個人的命運弧」是**人物層**反編年體；Scene-List-Scene 是**結構層**反均勻。兩條合起來 = AI 孢子從「事件 timeline」升級成「有弧線有呼吸」的 prose。

### 結構性題目用問題入口（v2.7，邦交國 #51/#52 教訓）

如果觸發 Hook Blueprint（政治/外交/制度/經濟結構性題目，見 [SPORE-VERIFY.md §Hook Blueprint](SPORE-VERIFY.md)），孢子開場預設用問句型。

**寫法公式**：

```
如果 / 明明 / 當 [讀者已知或可感的事實 A]，
為什麼 [與 A 衝突、但真實存在的事實 B]？
```

**邦交國實戰**：

- ❌ `台灣通往世界的路，沒有跟著縮成 12 條。`（抽象結論，缺少讀者可抓住的物件）
- ✅ `如果台灣只剩 12 個邦交國，為什麼你拿著護照，還能走進 177 個國家與地區？`（護照 = 日常物件；12 vs 177 = 矛盾；文章能回答）

**自檢三題**：

- 這句話裡有讀者能想像自己正在做的動作嗎？
- 這個問題是否讓人真的想知道答案，而不是只是在要求表態？
- 如果刪掉數字，場景還存在嗎？不存在 → 只是 briefing，要回 [SPORE-VERIFY §Hook Blueprint](SPORE-VERIFY.md) 重找物件。

## 模板庫

四種模板覆蓋所有文章類型。每篇孢子選最合適的一種。

### A. 人物型（最常用）

#### A1 標準版結構

```
你知道嗎？[emoji]
[時間]，[身份落差的起點]。

[具體場景 × 2-3 個]

[轉折：代價/賭注]

[結果：用數字說話]

[情感收尾：一句話]

完整故事 👉 <URL>
```

**A1 適合**：有名有姓的人、有完整故事弧線（起點→代價→結果）、有數字階梯

#### ✅ A1 成功範例：齊柏林

```
你知道嗎？🚁
1990 年，一個交通部公務員第一次搭上直升機，他的工作是拍國道施工照片。但當直升機升到 500 英尺，他看見的不只是公路——是蜿蜒的海岸線、層疊的山脈，還有平地上看不見的傷痕。

他開始「搭便機」。氣象局換班時跟著飛、林務局噴農藥時跟著飛、台電吊掛工程時跟著飛。只要有人上天，他就背著相機跟。

後來他拿出 300 萬積蓄，抵押房子借了 600 萬，買了 3000 萬的空拍設備。妻子說他瘋了。

三年後，這部紀錄片票房 2.2 億，是台灣影史最賣座的紀錄片。

他叫齊柏林。2017 年為續集勘景時墜機殉職，享年 52 歲。他身邊散落著百萬攝影器材——對他來說，那些畫面比生命更珍貴。

完整故事 👉 https://taiwan.md/people/%E9%BD%8A%E6%9F%8F%E6%9E%97/
```

**為什麼好**：身份落差開場（公務員→導演）、三個搭便機場景有畫面感、300→600→3000→2.2 億數字階梯、情感收尾不說教

#### A2 首尾呼應變體（v1.2，李洋 #28 萃取）

適合**多 angle 混合 + 時間跨度大**的人物。

##### A2 結構

```
[起手式] + [一個少年/孩子/早期版本的具體畫面，足夠細節能成為「icon」]

[早期版本的自白引語 — 那個版本對自己說的話]

[時間錨定加速：X 年後 → 成就 → 現在的職位]

[最新熱點 / 近期事件 — 高光時刻]

[反爆 / 數字嵌入故事 — 把光環拉回凡人]

[計算反差 — 讓讀者大腦自動算]

[沉默自白 — 一句話 + 場景化交代來源]

[首尾呼應 — 開場那個畫面以稍變形式回來]

完整故事 👉 <URL>
```

##### A2 核心技術

- **首尾呼應**：開場留下一個畫面（不只是場景描述，是讀者能在腦中「看見」的 icon），結尾讓那個畫面以稍變形式回來。**這是 A2 的核心技術**。
- **時間錨定**：用「X 歲」「Y 年後」「昨天」「今天」推進敘事，把多個 angle 串成一條時間軸。
- **數字嵌入故事**：B angle 的數字反差不單獨成段，要嵌進故事弧線的高光時刻。
- **沉默轉接**：兩個 angle 之間用一句話 / 一個畫面轉接，不用「另外」「同時」這類連接詞。
- **場景化引語**：「他只在 [臉書/訪談/頒獎台] 留了一句話：『XXX』」而非「『XXX』他說」。

##### A2 適合

- 有時間跨度的人物（少年→現在）
- 想呈現「過去那個版本仍然在他身上」的詩意
- 素材橫跨多個 angle，單一 angle 不夠抓核心矛盾
- 配合即時新聞熱點（24-48 小時內事件）
- 觀察者要求「混合策略 + 故事弧線」

#### ✅ A2 成功範例：李洋（孢子 #28，2026-04-14）

```
你知道嗎？🏸

14 歲，李洋進台北中山國中羽球班。每天早上 5 點半起床，媽媽騎機車載他到南勢角站趕首班捷運，轉 3 班車才到學校。

校門還沒開，他就先到學校旁的超商，補寫前一天沒寫完的作業。

他後來說：「體育班 15 個人，我屬於後面那一掛，齊麟是前面那一掛。」

那個後面那一掛的少年，等了快 9 年才跟齊麟搭上檔。東京、巴黎兩面奧運金，男雙史上首對二連霸。30 歲，史上最年輕的內閣部長。

昨天他把奧會補頒的一千萬獎金全部捐了。今天監察院公布他的財產申報：存款 3,401 萬，債務 3,638 萬。債務比存款多 237 萬，30 歲房貸族。

那一千萬，剛好是他存款的三成。

他只在臉書留了一句話：「現在我更在意的是制度與環境。」

14 歲那個在超商補作業等校門開的少年，仍然在他身上。

完整故事 👉 https://taiwan.md/people/%E6%9D%8E%E6%B4%8B/
```

### B. 冷知識型（適合文化/地理/科技）

#### 結構

```
你知道嗎？[emoji]
[反直覺事實，一句話]

[展開：為什麼會這樣]

**適合**：有一個「哦？」的驚奇點、概念性主題（不是單一人物故事）

#### ✅ B 成功範例：數位身分證

```
💳 你知道台灣花 48 億做了一張身分證，結果一張都沒發出去嗎？

完整故事 👉 https://taiwan.md/technology/...
```

**為什麼好**：荒謬數字開場（48 億→0 張）、政府 vs 民間對比翻轉、引語收尾有餘韻

```
你知道嗎？[emoji]
[一組讓人停下來的數字]

[這個數字的起點（越卑微越好）]

[中間的轉折/代價]

[結果：跟讀者的關聯]

完整故事 👉 <URL>
```

**適合**：數字本身就震撼、有明確的「從 A 到 B」跨度

### D. 時間軸型（適合歷史事件）

#### 結構

```
你知道嗎？[emoji]
[精確時間 + 地點]，[一件事發生了]。

[那之前是什麼狀態]

[事件本身的細節（場景感）]

[漣漪效應：這件事如何改變了...]

完整故事 👉 <URL>
```

**適合**：有明確歷史時刻、事件本身有戲劇性、影響深遠

### E. 串文型（適合深度故事 / Threads 串文）

> 4 則一組，每則 ≤ 150 字，合起來 = 完整弧線。適合素材豐富、一則裝不下的故事。

#### 結構

```
【第 1 則：鉤子】（獨立存活，讓人想往下滑）
[起手式] + [最強的反直覺事實或場景]
→ 這一則單獨截圖也要有傳播力

【第 2 則：展開】（背景脈絡）
[為什麼會這樣？把故事推回起點]
→ 補上讀者需要的背景，但不能變百科

【第 3 則：轉折】（代價/衝突）
[事情不是那麼簡單——代價、阻力、意外]
→ 這一則是情感高點

【第 4 則：收尾】（情感 + 連結）
[結果 + 餘韻收尾]
完整故事 👉 <encoded-URL>
```

#### 規則

- 每則 ≤ 150 字（硬上限）
- 第 1 則必須獨立存活 — 只看第 1 則也要有價值
- 4 則 = 完整弧線（鉤子→背景→衝突→收尾），不是把長文切四段
- 連結只放第 4 則
- 第 2-3 則結尾要有「往下滑的引力」，但不能用廉價 cliffhanger

```

## 多版本提案（觸發條件 + 流程）

> 當素材豐富到一個 angle 不夠時，預設產出 3 個 angle 版本給觀察者選。
>
> **v2.5 校準**：3-angle 不是**所有**素材豐富時的預設，是**「切角本身是關鍵決策」時**的預設。

### 觸發條件判準樹

```
素材豐富時，問一題：
「觀察者的真正決策是『選角度』還是『選語感』？」
  ├─ 選角度（多元 focal point 互斥）
  │   例：李洋「童年場景 vs 財產申報數字」
  │       鄭麗文「19 歲絕食 vs 政治轉向」
  │   → 走 3-angle 流程（保留切角選擇空間）
  │
  └─ 選語感（角度已明、弧線已明、需要 prose 手感微調）
      例：謝德慶 21 年一個人六件作品
          嚴長壽一生服務業
      → 直接綜合一版（Scene-List-Scene）+ 觀察者 prose 層微調
```

**自檢一問**：「這個人物有 2+ 個**互斥** focal point，還是 1 個弧線可以吞下所有高光？」

- 互斥 focal point → 3-angle
- 單弧 → 直接綜合

### 為什麼是預設步驟

李洋孢子（#28）實戰證明：人物素材豐富時，單一 angle 會漏掉重要面向。3 個 angle 版本各自獨立成立 + 加「我的推薦」，觀察者選定後再進混合策略，比單版本上呈品質高出一個等級。

**反例**（謝德慶 #39）：把 Cage / Time Clock / 21 年長弧切成三 angle 上呈，觀察者看完說「語感不太順，要更故事感、場景感一些，你直接綜合提出一個最好的版本」。根因：這三「angle」其實是**同一條弧線上的三個段落**，不是互斥切角。強行切三個讓每個都變成列表式摘要。綜合一版用 Scene-List-Scene 一次到位。

### 執行流程

1. **從萃取素材中找出 3 個最強 angle**。每個 angle 聚焦一個核心衝突 / 一組數字落差 / 一個情感支柱
2. **每個 angle 寫一個獨立 ≤300 字版本**。每個版本自己能單獨上線，不依賴其他版本存在
3. **每個版本附「鉤子三要素檢核」+「適合什麼情境」**（例如「最即時 / 最有畫面 / 最有跨度」）
4. **加「我的推薦」+ 推薦理由**。不是替觀察者決定，是給選擇判斷依據（觀察者可 override）
5. **觀察者選定後**：
   - 選單一版本 → 跳 SHIP stage
   - 要求兩個或多個版本混合 → 跳混合策略（下方）
   - 要求微調 → 改順後重呈

### Angle 命名建議

依「最強元素」命名，方便觀察者快速判斷：

| Angle 類型 | 範例命名                              | 適合素材                    |
| ---------- | ------------------------------------- | --------------------------- |
| 詩意對照型 | 「14 歲清晨四點 × 30 歲清晨四點」     | 有時間跨度 + 場景對照       |
| 數字反差型 | 「兩千萬剛好是存款的三成」            | 有強烈數字反差 + 即時新聞點 |
| 跨度型     | 「『你不是打羽球的料』→ 雙金 + 部長」 | 有童年到現在的長弧線        |
| 引語型     | 「『離開羽球場後，你還剩下什麼？』」  | 有核心提問可貫穿全文        |

## 混合策略：故事弧線串接

> 當觀察者選 A+B 或 A+C 混合時，**不是把兩段並列**。是把兩個 angle 用時間軸 / 因果鏈串成一條故事弧線。

### 為什麼這條規則存在

李洋孢子 #28 實戰：觀察者說「我要 A+B，我要有故事的」。第一輪寫成兩段並列被打回。**「我要有故事的」= 觀察者要結構性弧線，不是要兩個 angle 的拼貼**。

### 四個技術要點

1. **首尾呼應**：開場用 A angle 的最強畫面（例如「14 歲清晨四點便利商店等天亮」），結尾讓那個畫面以稍變的形式回來（例如「14 歲那個在便利商店等天亮的少年，仍然在他身上」）。**這是混合策略的核心技術**。
2. **時間錨定**：用「14 歲」「9 年後」「昨天」「今天」這類時間錨點推進敘事。把多個 angle 的時間戳串成一條時間軸。
3. **數字嵌入**：B angle 的數字反差**不單獨成段並列**。要嵌進故事弧線的高光時刻（例如「然後昨天，他把兩千萬全部捐了。今天他第一份財產申報公布⋯⋯」）。
4. **沉默轉接**：兩個 angle 之間用一句話 / 一個畫面轉接，**不用「另外」「同時」「除此之外」**這類連接詞。連接詞 = 並列；沉默 = 弧線。

### 錯誤示範 vs 正確示範

❌ **錯誤（並列式）**：

```
14 歲那年，李洋每天清晨四點搭四條捷運⋯⋯[A angle 完整段落]

另外，他今天的財產申報公布：存款 3,401 萬⋯⋯[B angle 完整段落]

兩個故事疊在一起，就是李洋。
```

✅ **正確（弧線式）**：

```
14 歲，李洋進了台北的中山國中羽球班⋯⋯[少年場景]

[時間錨定加速：9 年後 → 雙金 → 部長]

昨天，他把兩千萬全部捐了。[B angle 高光]

今天他第一份財產申報公布：[數字嵌入故事]

那兩千萬，剛好是他存款的三成。[計算反差]

[沉默轉接 + 自白]

14 歲那個在便利商店等天亮的少年，仍然在他身上。[首尾呼應]
```

### 自檢清單

- [ ] 開場有一個具體畫面 / 場景嗎？
- [ ] 結尾有讓開場那個畫面以稍變形式回來嗎？
- [ ] 中間有時間錨點推進嗎？（不是空間並列）
- [ ] 兩個 angle 之間沒有「另外」「同時」這類連接詞嗎？
- [ ] 數字嵌進故事段落而不是獨立並列段嗎？
