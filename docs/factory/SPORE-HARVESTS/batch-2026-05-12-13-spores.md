### 2. 台積電 X edit history（#69 `2053100425730269544` → `2053101189034860856`）

### 3. 無人機 X data error（#71 URL 內容實為台積電）

## 第一性原理 — dry-run 揭露的 routine v2.2 真實邊界

1. **Chrome MCP pairing 是 unattended cron 第一風險**：本 dry-run 第一次 `list_connected_browsers` 回 `[]` — 哲宇 in-loop 開 Chrome 才 pair 成功。明早 07:04 unattended cron 跑時，如果 Chrome / Mac 睡眠 → routine abort。`select_browser` 持久化但需 active browser
2. **Per-spore harvest cost**：每條 spore navigate + wait + read ~10s。13 spores = ~3 min routine wall-clock。可接受
3. **X edit history silent drift**：3/7 X URLs (43%) 是 deprecated edit targets — SPORE-LOG `url` 欄位需要 X edit history 同步機制，否則 routine 每次都 harvest 0 views/0 engagement 的 stale URLs
4. **Step 1-7 full pipeline 太重不適合 daily routine**：D+1-D+7 每天跑「完整 Step 1-7」是 SPORE-HARVEST canonical 但 categorize + 事實驗證 + 整合 是高 judgment cost，realistic routine 只能跑 Step 1 + 5 + 7（surface metrics + batch log），深度整合留 observer ad-hoc
