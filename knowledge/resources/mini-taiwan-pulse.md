# Mini Taiwan Pulse — 台灣交通即時 3D 視覺化 🌐

> 📖 **深度版文章**：這則資源已升級為公民科技深度研究文章，完整版請看 [Mini Taiwan Pulse：一個資料分析師如何把台灣的交通脈動畫成會呼吸的 3D 光軌](/technology/mini-taiwan-pulse)（2026-04-19）。本頁保留作為資源清單的索引條目。

> **30 秒概覽：** 一個將台灣交通運輸即時動態化為 3D 光球與光軌的開源專案。航班在天空劃出弧線、船舶在海面留下拖尾、列車在軌道上奔馳——23 個可切換圖層，讓你「看見」台灣的脈搏。

## 為什麼值得關注

大多數人看台灣地圖，看到的是靜態的輪廓。Mini Taiwan Pulse 讓你看到的是一座**正在呼吸的島嶼**。

這個專案的野心不小：把散落在各政府機關的開放資料——航班、船舶 AIS、台鐵高鐵時刻表、捷運路線、人口統計、氣象觀測——全部整合到同一張 3D 地圖上。不是簡單的點點標記，而是用光球、光軌、彗尾拖尾等視覺語言，把資料變成會動的風景。

> **📝 策展人筆記**
> 台灣的開放資料基礎建設在亞洲名列前茅（[全球開放資料指數](https://index.okfn.org/)多次進入前十），但「資料開放」和「資料被看見」之間存在巨大鴻溝。Mini Taiwan Pulse 正在填補這個缺口。

## 三層脈動

### 天空 — 航班光軌 ✈️

涵蓋全台 14 座機場、1,500+ 航班的即時動態。每架飛機是一顆發光的球體，身後拖著彗尾狀的漸層光軌。高度誇張倍率可調（1x～5x），讓低空航線和高空航線的差異一目了然。

資料來源：FlightRadar24 API。

### 海洋 — 船舶追蹤 🚢

台灣周邊海域的船舶位置，用青藍色光球標示，每艘船留下 30 分鐘的拖尾軌跡。系統自動過濾 GPS 異常跳躍和無效 MMSI，確保你看到的每一個光點都是真實的船。

資料來源：AIS（自動識別系統）船舶位置資料。

### 大地 — 六大軌道系統 🚄

這可能是最令人驚艷的部分。六個軌道系統同步運行：

| 系統             | 規模                                    |
| ---------------- | --------------------------------------- |
| 台鐵（TRA）      | 265 條路線、333 列火車，依車種 6 色分類 |
| 高鐵（THSR）     | 南北主線 + 支線                         |
| 台北捷運（TRTC） | 8 條路線                                |
| 高雄捷運（KRTC） | 紅線 + 橘線                             |
| 高雄輕軌（KLRT） | 環狀輕軌                                |
| 台中捷運（TMRT） | 綠線 + 藍線                             |

台鐵的處理特別複雜——OD 軌道匹配、彰化三角線這種分歧路線，都有專門的引擎處理。

資料來源：公開時刻表 + [OpenStreetMap](https://www.openstreetmap.org/) 軌道資料。

## 不只是交通

除了移動中的運具，專案還疊加了多層靜態與分析圖層：

- **基礎設施**：14 座機場邊界、535 座車站光柱（高度 = 停靠次數）、36 座燈塔的 3D 旋轉光束
- **路網**：國道（紅）、省道（橘）、自行車道（綠），zoom 自適應寬度
- **人口分析**：H3 六角格人口熱力圖，支援日間/夜間人流切換，9 項人口指標
- **氣象**：觀測站即時資料 + 溫度波浪 3D 曲面（0.03° 格點解析度）
- **新聞**：CNA 中央通訊社 RSS + Gemini API 地理編碼，把新聞事件標在地圖上
- **國道壅塞**：即時壅塞程度色彩編碼

總計 **23 個可獨立切換的圖層**，十個分類。

## 技術亮點

- **TypeScript + Mapbox GL + Three.js**：2D 地圖用 Mapbox 原生渲染，3D 元素（光球、光軌、光柱、溫度曲面）用 Three.js 疊加
- **效能考量**：船舶用 InstancedMesh 批次渲染、視口剔除（viewport culling）避免渲染看不到的物件
- **色彩科學**：人口圖層用 Plasma / Viridis / Inferno 等感知均勻色階，log1p + gamma 正規化處理重尾分布，色盲友善
- **MIT 授權**：完全開源，歡迎 fork 和貢獻

> **📝 策展人筆記**
> 用 additive blending 做光軌疊加是個聰明的選擇——多條航線重疊的區域自然變亮，視覺上就能看出航路的繁忙程度，不需要額外的統計圖表。

## 開放資料生態

這個專案串接的資料來源，本身就是一份台灣開放資料的導覽清單：

| 資料               | 來源                                              |
| ------------------ | ------------------------------------------------- |
| 航班即時位置       | FlightRadar24 API                                 |
| 船舶 AIS           | 國際船舶自動識別系統                              |
| 鐵道時刻           | 公開時刻表 + OSM                                  |
| 公車/客運/自行車   | [TDX 公共運輸資料](https://tdx.transportdata.tw/) |
| 人口統計           | [SEGIS 統計地理資訊](https://segis.moi.gov.tw/)   |
| 氣象觀測           | [中央氣象署](https://www.cwa.gov.tw/)             |
| 離岸風場           | 經濟部能源局                                      |
| 新聞事件           | 中央通訊社 CNA RSS                                |
| 機場/港口/車站邊界 | [OSM Overpass API](https://overpass-turbo.eu/)    |

⚠️ **值得注意的是：** 台灣的 [TDX 運輸資料流通服務](https://tdx.transportdata.tw/) 是少數將全國公共運輸資料統一標準化的政府平台，涵蓋公車、客運、鐵道、自行車等，API 文件完整且免費使用。這在全球範圍內並不常見。
