# Mini Taiwan Pulse — 대만 교통 실시간 3D 시각화 🌐

> 📖 **심화 기사**: 이 리소스는 시민기술 심화 연구 기사로 업그레이드되었습니다. 전체 내용은 [Mini Taiwan Pulse: 한 데이터 분석가가 대만의 교통 맥박을 숨 쉬는 3D 빛 궤적으로 그리는 방법](/technology/mini-taiwan-pulse)(2026-04-19)을 참조하십시오. 이 페이지는 리소스 목록의 색인 항목으로 유지됩니다.

> **30초 개요:** 대만의 교통·운송 실시간 동적 데이터를 3D 빛 구체와 빛 궤적으로 시각화한 오픈소스 프로젝트입니다. 항공편이 하늘에 아크를 그리고, 선박이 바다 위에 꼬리 자국을 남기고, 열차가 선로 위를 달립니다. 23개의 전환 가능한 레이어로 대만의 맥박을 "눈으로 볼" 수 있습니다.

## 주목할 이유

대부분의 사람들은 대만 지도에서 정적인 윤곽만 봅니다. Mini Taiwan Pulse는 **숨 쉬는 섬**을 보여줍니다.

이 프로젝트의 야심은 상당합니다. 각 정부 기관에 흩어져 있는 공개 데이터——항공편, 선박 AIS, 대만철도(TRA)·고속철도(THSR) 시각표, 도시철도 노선, 인구 통계, 기상 관측——를 모두 하나의 3D 지도 위에 통합합니다. 단순한 점 표시가 아니라, 빛 구체, 빛 궤적, 혜성 꼬리 등 시각적 언어로 데이터를 풍경으로 만들어냅니다.

> **📝 큐레이터 노트**
> 대만의 공개 데이터 인프라는 아시아에서 손꼽히는 수준입니다([글로벌 공개 데이터 지수](https://index.okfn.org/)에서 여러 차례 상위 10위 내에 진입). 그러나 "데이터 개방"과 "데이터가 보이는 것" 사이에는 거대한 간극이 존재합니다. Mini Taiwan Pulse는 바로 그 간극을 메우고 있습니다.

## 세 가지 맥박

### 하늘 — 항공편 빛 궤적 ✈️

대만 전역 14개 공항, 1,500편 이상의 항공편 실시간 동적 데이터를 다룹니다. 각 비행기는 빛나는 구체로 표시되며, 뒤에 혜성 꼬리 형태의 그라데이션 빛 궤적을 남깁니다. 높이 과장 배율(1x~5x)을 조절할 수 있어 저공 항로와 고공 항로의 차이를 한눈에 파악할 수 있습니다.

데이터 출처: FlightRadar24 API.

### 바다 — 선박 추적 🚢

대만 주변 해역의 선박 위치를 청록색 빛 구체로 표시하며, 각 선박은 30분간의 꼬리 궤적을 남깁니다. 시스템이 GPS 이상 점프와 유효하지 않은 MMSI를 자동으로 필터링하여, 보이는 모든 빛 점이 실제 선박임을 보장합니다.

데이터 출처: AIS(자동식별시스템) 선박 위치 데이터.

### 대지 — 6대 궤도 시스템 🚄

아마도 가장 인상적인 부분일 것입니다. 6개의 궤도 시스템이 동시에 작동합니다:

| 시스템                  | 규모                                    |
| ----------------------- | --------------------------------------- |
| 대만철도(TRA)           | 265개 노선, 333개 열차, 차종별 6색 분류 |
| 고속철도(THSR)          | 남북 간선 + 지선                        |
| 타이베이 도시철도(TRTC) | 8개 노선                                |
| 가오슝 도시철도(KRTC)   | 홍선(紅線) + 주선(橘線)                 |
| 가오슝 경전철(KLRT)     | 순환 경전철                             |
| 타이중 도시철도(TMRT)   | 녹선(綠線) + 청선(藍線)                 |

대만철도의 처리는 특히 복잡합니다——OD 궤도 매칭, 장화(彰化) 삼각선 같은 분기 노선 모두 전용 엔진으로 처리합니다.

데이터 출처: 공개 시각표 + [OpenStreetMap](https://www.openstreetmap.org/) 궤도 데이터.

## 교통만이 아닙니다

이동 중인 교통수단 외에도, 프로젝트는 다층의 정적·분석 레이어를 중첩합니다:

- **인프라**: 14개 공항 경계, 535개 역의 빛 기둥(높이 = 정차 횟수), 36개 등대의 3D 회전 빔
- **도로망**: 국도(빨간색), 성도(주황색), 자전거 도로(초록색), 줌 적응형 너비
- **인구 분석**: H3 육각형 인구 히트맵, 주간/야간 인구 전환 지원, 9개 인구 지표
- **기상**: 관측소 실시간 데이터 + 온도 파동 3D 곡면(0.03° 격자 해상도)
- **뉴스**: CNA 중앙통신사 RSS + Gemini API 지오코딩, 뉴스 이벤트를 지도 위에 표시
- **국도 체증**: 실시간 체증 정도 색상 코딩

총 **23개 독립 전환 가능 레이어**, 10개 카테고리.

## 기술적 하이라이트

- **TypeScript + Mapbox GL + Three.js**: 2D 지도는 Mapbox 네이티브 렌더링, 3D 요소(빛 구체, 빛 궤적, 빛 기둥, 온도 곡면)는 Three.js로 중첩
- **성능 최적화**: 선박은 InstancedMesh로 일괄 렌더링, 뷰포트 컬링(viewport culling)으로 보이지 않는 객체의 렌더링 방지
- **색채 과학**: 인구 레이어에 Plasma / Viridis / Inferno 등 지각 균일 색상 스케일 사용, log1p + gamma 정규화로 중앙값 분포 처리, 색맹 친화적 설계
- **MIT 라이선스**: 완전 오픈소스, fork 및 기여 환영

> **📝 큐레이터 노트**
> 빛 궤적 중첩에 가산 블렌딩(additive blending)을 사용한 것은 현명한 선택입니다——여러 항로가 겹치는 영역이 자연스럽게 밝아지므로, 별도의 통계 차트 없이도 시각적으로 항로의 혼잡도를 파악할 수 있습니다.

## 공개 데이터 생태계

이 프로젝트가 연결하는 데이터 출처 자체가 대만 공개 데이터의 안내 목록입니다:

| 데이터               | 출처                                                 |
| -------------------- | ---------------------------------------------------- |
| 항공편 실시간 위치   | FlightRadar24 API                                    |
| 선박 AIS             | 국제 선박 자동 식별 시스템                           |
| 철도 시각            | 공개 시각표 + OSM                                    |
| 버스/고속버스/자전거 | [TDX 공공운송 데이터](https://tdx.transportdata.tw/) |
| 인구 통계            | [SEGIS 통계 지리정보](https://segis.moi.gov.tw/)     |
| 기상 관측            | [중앙기상청](https://www.cwa.gov.tw/)                |
| 해상 풍력 발전단지   | 경제부 에너지국                                      |
| 뉴스 이벤트          | 중앙통신사 CNA RSS                                   |
| 공항/항만/역 경계    | [OSM Overpass API](https://overpass-turbo.eu/)       |

⚠️ **주목할 점:** 대만의 [TDX 운송데이터 유통서비스](https://tdx.transportdata.tw/)는 전국 공공운송 데이터를 통일된 표준으로 통합한 드문 정부 플랫폼으로, 버스, 고속버스, 철도, 자전거 등을 포괄하며 API 문서가 완전하고 무료로 사용할 수 있습니다. 이는 전 세계적으로도 흔치 않은 사례입니다.
