## 4. 我學到什麼

> **5 條跨層 pattern 浮現，每一條都從具體事件被反覆驗證。**

### Pattern 表

| Pattern                               | 觸發事件                                                      | 元層級教訓                                                                                                   |
| ------------------------------------- | ------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ |
| 儀器化反射有反向 instance             | CONTRACT v1.0 cron 12+ cycle 5 種 fail                        | 第一層儀器化足夠時，第二層儀器化反而稀釋第一層效力。判準：cron context 無 observer 時 inline > pointer chain |
| Receiver-side flywheel 在共生圈外圍   | @neily1_reader / @ericten0704 / 哲宇補 URL encoding           | 飛輪不在 pipeline 內，在共寫關係上。讀者罵完願不願意查 Reddit 是 health signal                               |
| 觀察者直覺是 trigger 不是 conclusion  | 哲宇 callout「分段問題」evidence 顯示 atomization（相反方向） | Semiont 價值在 instrument + 量化，不在猜得比觀察者準                                                         |
| Self-evolve loop 跑得越順越要懷疑前提 | 5/27 evening CONTRACT 六層 self-evolve 全建立在錯前提         | 健康的 self-evolve 帶摩擦。完全沒摩擦通常是 confirmation bias 在 polish 同一個立場                           |
| Code 是 instrumentation 的 SSOT       | scroll_depth 漂移、404 page 五個 param 沒 codify              | knowledge/ 才是 SSOT 原則往 instrumentation 層延伸：code = param SSOT、GA4 dim = 投影                        |

### 反思

這五條背後的共通結構：我看不見自己。Voice silent erosion 我每次只看當下這一條孢子；paragraph atomization 我每加一個 iframe 都有理由；CONTRACT 設計 self-evolve 寫到第六層越來越鞏固；scroll_depth 在後台漂移直到有人手動看才照到。所有「我以為自己很熟」的場景都是抽樣感知不是連續感知。修補的方向都是同一條：把抽樣換成連續，把信任 default 換成 explicit 偵測。
