# Taiwan.md 22 縣市系列文章 — 規劃與初步調查

> Taiwan.md 目前 Geography 類別有總覽性條目（《城市特色與區域文化》《台灣行政區劃》《台灣都市發展與城鄉差距》），但**沒有任何一篇是單一縣市的策展性深度文章**。本規劃補上這個結構性缺口。
>
> 22 縣市文章不是百科條目（維基百科已經做得很好），是**「住在這裡的人會說的版本」**——MANIFESTO §策展非百科的縣市層落實。

---

## 1. 為什麼這個系列

**SEO + AI 引用雙重缺口**：
- Google 搜「基隆 / 嘉義 / 雲林 / 連江」這類縣市名加台灣關鍵字，第一頁多是政府官網（公文體）、旅遊網站（觀光手冊）、維基百科（百科體）— 沒有 first-person curatorial voice
- AI agent（GPTBot / Perplexity / ChatGPT-User）回答「Tell me about Chiayi」類查詢，缺乏 Taiwan-perspective canonical article = AI 改用維基或外文 source

**Taiwan.md 策展角度的獨特性**：
- 每個縣市都有「外人覺得」vs「住在這裡的人覺得」的張力
- 行政區劃只是骨架，**人文紋理才是肉**（地方政治派系 / 產業興衰 / 地理決定的生活方式）
- 政治經濟史的縣市層敘事（彰化埋葬日本兵的山 / 宜蘭五結反公投 / 雲林台塑六輕 / 屏東港口轉型）很少被串成連貫故事

**系列效應**：22 篇互相 cross-link 形成「台灣地理閱讀地圖」，類別頁 `/geography/` 整體 authority 拉升。

## 3. 縣市文章共通結構模板（草案 v0.1）

每篇縣市文章建議走以下 7 個 H2 骨架。Pilot 跑完後校準此 template。

```
# {縣市}：{副標 hook}

> **30 秒概覽：** {人口/位置/最大產業/最強標誌}，但這篇文章想說的是 {核心矛盾}。

## 地理決定的命運
- 自然環境（山 / 海 / 平原 / 氣候）如何形塑生活方式
- 跟鄰近縣市的關係（依賴 / 競爭 / 漏接）

## 從清領到 2026 的軸線（3-5 個轉折點）
- 不寫編年體（per chronicle-lead plugin），用 3-5 個關鍵時刻串
- 清領 / 日治 / 戰後 / 解嚴後 / 21 世紀，每段一個 anchor 事件

## 經濟肌理
- 在地產業的歷史（什麼起家、什麼衰落、什麼接棒）
- 跟政治派系 / 中央政策的關係
- 移工 / 移民 / 在地人的勞動結構

## 文化標誌與在地識別
- 食物（必選 1-3 項代表 + 跟其他縣市同名食物的差異）
- 節慶 / 信仰中心 / 方言腔調
- 一首歌 / 一部電影 / 一個導演把這個地方寫進記憶

## 不被看見的角落（curatorial 角度）
- 觀光手冊不會寫的部分：派系 / 環境代價 / 人口外流 / 高齡化
- 在地人會說但外人不知道的故事
- 政治史的縣市層敘事（不是中央級事件，是這個縣市自己的政治史）

## 在地人會帶你去的 3 個地方
- 不是觀光地標，是「我帶你去吃 / 看 / 走」的 3 個 anchor
- 每個地點配一段散文（不是清單條列）

## 延伸閱讀
- cross-link 既有 Taiwan.md 文章（食物 / 人物 / 文化 / 歷史）
- cross-link 鄰近縣市文章（系列效應）
```

**篇幅目標**：每篇 4500-6000 CJK chars（depth article threshold），15-25 footnotes。

## 4. 共通 research caveats（22 entries 統一加進 Notes）

縣市文章研究時容易踩的坑（pilot 跑完後再 enrich）：

1. **人口/面積數字會變動**：以內政部戶政司「最新月報」為 source，引用時標日期（例：「截至 2026 年 X 月 X 日」）。不寫「目前約」這種會 rot 的描述
2. **行政沿革複雜**：直轄市改制（桃園 2014 / 台南 2010 / 新北 2010 / 台中 2010 / 高雄 2010 縣市合併）— 寫到「縣」「市」要釐清是哪一年的稱呼
3. **族群結構不能簡化**：原住民族別不是「原住民」一詞蓋過去（卑南 vs 排灣 vs 阿美 vs 太魯閣 vs 賽夏 各有 specific 文化）。客家 vs 閩南 vs 外省也要 specific
4. **產業興衰要驗證**：「以前 X 很有名」這種敘事必須三源驗證（公司歷史 / 報導 / 在地誌）
5. **地方政治派系**：黑金 / 派系械鬥 / 地方家族（如雲林林家、彰化謝家、屏東潘家、花蓮傅家）要寫但要避免泛化敘事 — 引用具體事件 + source
6. **食物名稱地方差異**：「肉圓」「米粉」「貢丸」「擔仔麵」每個縣市做法不同 — 不寫「全台最好吃」這種主觀宣稱
7. **觀光手冊腔調**：「美麗的 X / 浪漫的 Y / 醉人的 Z」一律刪掉，per EDITORIAL §塑膠句禁用
8. **地震/災害敘事**：921（南投）/ 0403（花蓮）/ 八八（屏東/高雄/台東）等災害事件需用具體傷亡/影響數字，不寫「重創 / 嚴重打擊」抽象詞
9. **眷村/移民史**：眷村文化 ≠ 整個外省人經驗（不同縣市眷村背景差異大：高雄左營海軍 vs 桃園陸軍 vs 新竹空軍）
10. **族群敏感引述**：原住民相關描述優先用部落自己的命名/說法（不是「番社」「山胞」這種殖民詞彙）— per EDITORIAL §族群書寫

## 5. Pilot 選擇建議

**推薦 pilot：基隆市（#7）**

**理由**：
- **規模適中**：人口 36 萬，研究範圍可控（vs 台北 247 萬 too much / 連江 1.3 萬 too niche）
- **獨特性極強**：雨都 + 北部唯一深水港 + 廟口夜市 + 礦業遺跡（瑞芳本屬基隆礦業圈）— hook 密度高
- **SC opportunity**：基隆相關 query「基隆 廟口 / 雨都 / 港口 / 八斗子」應該有 query volume，但 Taiwan.md 目前空白
- **可重用性高**：基隆走完 7 個 H2 骨架後，可以校準 template 給其他 21 縣市
- **跟既有文章 cross-link 機會多**：金瓜石 / 野柳已存在，可雙向連結
- **政治敏感度低**：相對中性題材，pilot 不會卡在敏感議題校準

**備選**：嘉義市（規模類似，雞肉飯 + 阿里山入口 + 北回歸線 hook 強，但跟阿里山小火車有歷史糾葛較複雜）

**避選 pilot**：
- 台北市 / 新北市 — 太大、資料海、容易迷失
- 連江縣（馬祖）/ 澎湖縣 — 太 niche、reader entry 點少
- 雲林縣 / 屏東縣 — 政治敏感度高（六輕、派系），pilot 容易卡在 framing

## 8. 風險與不確定性

1. **取材 source 在地性**：很多縣市的細節在「在地誌」「縣志」「鄉鎮誌」這類非數位化資源，網路 source 不足以支撐 depth article — 可能需在地 contributor 介入或實地調查
2. **「策展角度」對縣市 vs 對人物的差異**：人物文章有清楚主角，縣市沒有單一主角 — 寫作難度更高，怕變成「景點清單 + 食物清單」百科腔
3. **族群/政治敏感校準**：雲林（六輕）/ 苗栗（破產）/ 屏東（潘家）/ 花蓮（傅家）這些敏感敘事如何避免成為政治評論
4. **SC ROI 不確定**：縣市文章流量可能不如人物文章（人物有粉絲 base / 即時新聞 amplification，縣市較長尾）
5. **22 篇是否真該全部做**：澎湖/金門/連江人口少，可能合併為「離島專輯」EVOLVE 既有條目即可，不必每個獨立 NEW

## 10. Pilot 跑完後 retrospective — 給其他 21 篇的 enriched caveats

> 基隆市跑完 Stage 0-5 後從實戰萃取的額外 caveats，補進 §4 共通 list 給其他 21 縣市 entry 用。

### 新增 caveats（11-15）

11. **降雨數字要分「年降雨量」vs「降雨日數」vs「最多雨測站」**：基隆「雨都」品牌是 brand truth 但實際數據複雜——年降雨量 3,589.6 毫米（維基），暖暖測站 2024 年總降雨 5,695 毫米全台平地第一，但七堵 230 天「僅排到全國第 37 名」（全國前三是花蓮秀林 334 / 宜蘭大同 320 / 花蓮萬榮 304）。寫降雨類「全國最 X」claim 必須明確標「年總雨量」or「降雨日數」or「平地」or「測站」layer，混為一談會被質疑。

12. **港口運量「超越」的年份精確度**：高雄港超越基隆港的年份是常被混用的事實——agent Stage 1 抓到的多源說「1960 年代起逐漸超越」，沒找到單一明確 1969 年「超越」的 verbatim source。寫類似結構性轉變（規模超越、產業崩解、政策替代）時，**用「逐漸」+ 區間時間**（1960 年代後期），不用單一年份斷言，除非有原始官方統計。對應 Step 1.4 fact verification 應更嚴格 cross-check 結構轉變類事實。

13. **「首都看不見」式 frame 必須包含具體數字證據**：城市衰退敘事很容易變成 generic 抱怨。基隆篇用「39% 跨縣市通勤」「26% 空屋率」「世界第 7 大貨櫃港 → 第 113」三組數字撐住「被看不見」的 framing。其他 21 縣市寫類似敘事（被遺忘的港都／衰退的傳產縣／人口外流的山區縣）必須對應準備同等級數字證據，不能只靠形容詞。

14. **本地 vs 觀光客視角的對位是 Geography 類型加權矩陣的核心**：基隆篇開場用「凌晨四點的崁仔頂」（本地）對位「廟口夜晚」（觀光），全篇以「在地人帶你去的角落」（正濱漁港 + 阿根納廢墟 + 主普壇）替代清單式景點。其他 21 縣市 Stage 0.6 §觀點成型 必須答清楚「外人記憶 vs 在地人記憶」的具體 fault line，找不到 = 觀點不夠成型。

15. **「日治時代是台灣現代化轉折」這個敘事在縣市層必須具體化**：基隆篇用「築港五期 1899-1935」+「1924 升格為市」+「1930 全台第三大都市」+「西二/西三碼頭倉庫 1932/1934」具體化日治影響。其他 21 縣市寫日治影響時要避免抽象「日治時期奠定現代化基礎」這類塑膠句，必須有具體建築/年份/數字。

### 22 entries 該補強的點（基隆 pilot 揭露）

- **直轄市 vs 省轄市 vs 縣的篇幅差異**：基隆 36 萬人寫了 5168 字。台北 247 萬人不能用同樣比例寫成 35,000 字——直轄市文章篇幅應跟「能說清楚的核心矛盾」對齊，不是跟人口比例。
- **每個縣市要找出自己的「凌晨四點時刻」**：不是字面凌晨四點，是「外人不知道但在地人覺得最迷人的時刻」。基隆是崁仔頂；台北可能是萬華早晨 5 點的廟口豆漿、台南可能是 6 點國華街早餐街、高雄可能是凌晨港邊的卸貨。Stage 0.6 §觀點成型 必須答出這個 anchor。
- **「跟更大故事的連結」必須明確**：基隆篇把基隆連到「台灣海洋國家身份」這個大敘事。其他 21 縣市要明確答：你這個縣市對「台灣是什麼」這個更大問題提供了什麼答案？（屏東 = 南島文化 / 雲林 = 農業集體記憶 / 連江 = 戰地遺產與兩岸距離 / etc.）
