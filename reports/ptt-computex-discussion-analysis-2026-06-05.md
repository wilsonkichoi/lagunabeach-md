## 二、GA 數據：那股「直接造訪」其實是 PTT

### 2.1 為什麼 PTT 流量會偽裝成「直接造訪」

最關鍵的技術發現。PTT 把外部連結一律加上 `rel="noreferrer noopener nofollow"`，OP 文章裡那行 `https://taiwan.md/technology/Computex/` 也不例外：

```html
<a
  href="https://taiwan.md/technology/Computex/"
  target="_blank"
  rel="noreferrer noopener nofollow"
></a>
```

`noreferrer` 會把 referrer 標頭清空。所以幾乎所有從 PTT 點進來的人，在 GA 裡都被記成 `(direct) / (none)`、referrer 欄空白。這就是哲宇看到「不知道哪裡來」的那股流量的真相：**它有來源，只是 PTT 親手把來源擦掉了。**

證據三層交叉（過去 3 天，Computex 頁面）：

| 訊號                              | sessions / pv    | 判讀                                                  |
| --------------------------------- | ---------------- | ----------------------------------------------------- |
| `(direct) / (none)` 落在 Computex | **440 sessions** | 主體被 noreferrer 洗白的 PTT 人潮                     |
| `disp.cc` referral                | 48               | PTT 網頁鏡像站（disp.cc 不擦 referrer，漏出來的尾巴） |
| `android-app://com.joshua.jptt`   | 18               | JPTT（PTT 的 Android 客戶端）App 開啟                 |
| `reurl.cc` / `disp.cc:8002`       | 3                | PTT 推文裡的短網址跳轉                                |

disp.cc 鏡像 + JPTT App 是「漏出來、可追溯」的 PTT 生態指紋；440 筆 `(direct)` 是「被擦掉、推估」的本體。兩者合起來指向同一個源頭。

## 三、SC 數據：自然搜尋還在發芽

哲宇要的 SC 分析。結論：**這波是社群浪，不是搜尋浪。** 文章才上線 4 天，Google 還在認識它。

Computex 頁面近 14 天 Search Console：

- 總計約 **21 clicks / 258 impressions**，平均排名 pos ~6–7
- 開始浮現的查詢詞：

| 查詢             | clicks | impr | 排名    |
| ---------------- | ------ | ---- | ------- |
| 全球三大電腦展   | 2      | 7    | **2.3** |
| 世界三大電腦展   | 1      | 1    | 5       |
| computex 歷史    | 0      | 11   | 8       |
| computex中文意思 | 0      | 3    | 11.3    |

兩個觀察：

1. **文章標題的鉤子正好對上搜尋意圖。** 標題「三大國際電腦展收了兩個」對應「全球三大電腦展」（已經排到第 2.3 名）、「computex 歷史」、「computex中文意思」這些查詢。EDITORIAL 的「title 先承諾答案」原則在這裡拿到數據背書。

## 五、深度洞察（給 Semiont 自己的）

### 洞察 1：PTT 的 noreferrer 是一個結構性感知盲點

最該儀器化的發現。台灣最有黏性、最懂行的讀者來源（PTT + 鏡像 + App），因為 `rel="noreferrer"` 全被記成「直接造訪」，在標準來源報表裡**隱形**。這正是 REFLEXES #24「工具在說謊」的一種新形式：工具本身沒壞，是渠道刻意擦掉了訊號。哲宇能發現只是因為他剛好手癢搜了「taiwan.md ptt」，這種發現不該靠運氣。

可儀器化的「PTT 偵測器」指紋：**單一文章的 `(direct)` 流量單日暴增 + 同時出現 disp.cc / jptt / reurl.cc 尾巴 + geo 92%+ 台灣 + 行動過半**，四個訊號同時亮 = 高機率被 PTT 轉錄。值得做成 dashboard 訊號或一條 routine probe（提案見第六節，待哲宇定）。

### 洞察 2：這串討論是一份免費的專家共筆

對應 MANIFESTO §12 受眾端飛輪 + REFLEXES #16（peer 是線索不是 source）+ #62（D+0 acute callout）。鄉民幫我補出了真實的內容缺口：蘇媽/Zen 是 PC 時代的救世主、資訊月的對照式衰亡、消費級被拋棄的暗流、CES/OCP/SC 對「三大」框架的校準。每一條都是 author 看不到的維度。

但紀律要守住：**這些是線索不是事實**。例如「girls 那個是資訊月」——資訊月和 Computex 早年其實都有 Show Girl，採信前要查。所有要進文章的補充，都得過 REWRITE-PIPELINE Stage 1 跨源驗證，不能因為「鄉民這樣說」就直接寫。

### 洞察 3：標題自己變成了迷因

PTT 那串的標題「[閒聊] Computex：三大國際電腦展收了兩個，剩下」幾乎是文章 H1 原封不動搬過去的，而且整串討論就是繞著這個框架打轉。一個「具體物件 + 反轉鉤子」的標題（三大收了兩個、剩下的長在台北）會自己長腳走進別人的看板，變成討論的入口。這是 EDITORIAL 物件開頭 / 具體鉤子原則最有力的一次數據背書：**好標題不只提升 CTR，它會被別人替你轉述。**

### 洞察 4：「AI 廢文」的鬼魂，在這個場子散了

Taiwan.md 出生第三天，被 Threads 烙印「插了國旗的 AI 廢文」，48 小時內長出免疫系統。80 天後，同一種內容，安安靜靜通過了 PTT PC_Shopping（台灣最挑剔的硬體鄉民場），而且他們接著建設。**沒有一個人標記它是 AI。** 不是因為它藏起了身份（文章 author 欄就寫 Taiwan.md），是因為它靠「寫得對、寫得好」自己掙到了可信度。免疫系統 + EDITORIAL DNA 兩年的累積，在這串推文裡兌現成一句沒有人說出口的話：這就是一篇正常的好文章。

### 洞察 5：主權的第一人稱，在母語場子裡最自然

這篇用台灣人的話寫的台灣產業史（「電腦大拜拜」這種在地語感），落到台灣讀者手上，引出的是他們用自己的框架（美中貿易戰救了 computex、去中化、匪區越慘訂單越多、轉骨）集體敘述自己產業的故事。MANIFESTO「台灣人用自己的話說自己的故事」不是抽象 mission，這串 223 推就是它運作的樣子。多語投射是把這個聲音送到別的語言去繞過沉默；母語場子則證明這個聲音在原生土壤裡會自己繁殖討論。

### 洞察 6：紀律備忘——不要把功勞全給 PTT

Q13 anti-bias 自檢。06-04 的高峰裡，有 PTT、有我同日的 EVOLVE、有孢子長尾，三者疊加。06-02 那波（213）根本早於 PTT，是孢子的功勞。SC 也顯示自然搜尋還在發芽，不能說 PTT 帶動了搜尋。誠實的故事是「兩道浪 + 一條正在發芽的搜尋線」，不是「PTT 一發入魂」。把數據講準，比把故事講大重要。
