# 首頁改版十天後：它到底有沒有變好

5/26 晚上 ship 的首頁大改版（ReaderDoors / OrganismPreview / 心跳條 / 13 section + 深層 instrumentation），D+2 watch 當時看到停留 +104%，但自己也標了一句但書：那是上線當天的新鮮感，要等 7-14 天才知道是不是真的留得住。今天是 D+10，這份報告就是來回答那句但書：**改版有沒有實際改善互動、停留、轉換。**

一句話結論：**有，而且是真的留住了，不是上線煙火。但「停留」這個詞要拆開講——讀者並沒有在首頁上停更久，是更多讀者被首頁說服「值得往下走」，於是整段 session 變深了。** 首頁從一個「看一眼就走的門面」變成「一個會把人推進站裡的發射台」。

---

## 一、怎麼比的（窗口與方法）

改版是 2026-05-26 23:00 ship 的。要回答「有沒有改善」就得比改版前後同樣位置（zh-TW 首頁 `pagePath=/`）的同樣指標。

| 窗口       | 期間              | 用途                                                         |
| ---------- | ----------------- | ------------------------------------------------------------ |
| **BEFORE** | 5/19–5/25（7 天） | 改版前 baseline                                              |
| ship       | 5/26 23:00        | Wave 1+2+3 上線                                              |
| 上線新鮮期 | 5/27–5/28         | D+2 watch 看的兩天，含 launch 煙火，本報告**不拿來代表常態** |
| **AFTER**  | 5/29–6/3（6 天）  | 過了新鮮期、GA4 attribution 已結算的穩態                     |

**三個刻意排除的雜訊**（誠實先講）：

1. **6/4–6/5 不採用**：GA4 attribution lag 還沒結算（engagementRate 掉到 0.2%、bounce 衝到 99%，是結算延遲的指紋，不是真值）。等同 D+2 報告當時 5/28 那筆 496s 的 lag inflation，不能信。
2. **5/27–5/28 上線煙火另計**：launch 當天有新鮮感曝光，不能代表常態。
3. **看「率」不看「量」**：BEFORE 窗口首頁 activeUsers 1873、AFTER 1377（少 26%）。流量本身有週變動（BEFORE 5/19–20 首頁有一波 411→598 的高點）。所以比較一律用**跟流量無關的率指標**（engagementRate / bounceRate / 每人停留秒數），不用絕對人次。

**一個限制要先承認**：on-page 的 instrumentation（section_view / scroll_depth / time_milestone / click）是**跟著改版一起 ship 的**，改版前沒有這層資料。所以那些 funnel 數據只能看「改版後好不好」，沒有 before/after 可比。能做 before/after 的是 GA4 原生的 session 指標（停留 / 互動率 / 跳出率）。

---

## 二、停留：要拆成兩個問題

這是最容易看錯的地方。如果只看 `averageSessionDuration`，會得到一個漂亮但會誤導的數字：

| 指標                                          | BEFORE 5/19-25 | AFTER 5/29-6/3 |         Δ |
| --------------------------------------------- | -------------: | -------------: | --------: |
| avgSessionDuration（session 級）              |            63s |       **275s** | **+334%** |
| 首頁本身 engagementDuration / 每人（page 級） |          16.1s |          18.9s |      +17% |
| 首頁本身 engagementDuration / 每 session      |          12.9s |          15.2s |      +18% |

**這兩個數字差了二十倍，差別就是答案本身。**

`avgSessionDuration` 是 session 級的——它算的是「碰過首頁的那整段 session 有多長」，包含後面點進去讀的每一篇文章。它翻了四倍。

`engagementDuration / 每人` 是 page 級的——讀者實際盯著首頁看的時間。它只從 16 秒變 19 秒，幾乎沒動。

合起來讀：**改版沒有讓人在首頁上多停。讓人多停的是整個 session——讀者在首頁停的時間差不多，但更多人從首頁「決定往裡面走」，於是後面讀文章的時間把整段 session 拉長了四倍。**

這比「首頁停留變四倍」這個 naive 讀法誠實得多，也更有用：首頁的工作本來就是把人送進站裡，不是把人黏在首頁停更久。資料說它把這件事做得好多了。

---

## 三、互動：engaged 比例幾乎翻倍，而且穩住了一週

這是最乾淨、最可信的證據（engagementRate / bounceRate 是 GA4 原生複合指標，對單一 outlier session 不敏感，比 avgDuration 穩）：

| 指標                                       | BEFORE 5/19-25 | AFTER 5/29-6/3 |                    Δ |
| ------------------------------------------ | -------------: | -------------: | -------------------: |
| **engagementRate**（engaged session 佔比） |            37% |        **64%** |             **+73%** |
| **bounceRate**（跳出率）                   |            63% |        **36%** | **-43%（近乎砍半）** |
| engagedSessions（絕對數）                  |            870 |          1,097 |                 +26% |

最有力的一句：**AFTER 窗口總用戶比 BEFORE 少 26%，engaged session 的絕對數量卻多了 26%。** 一邊流量在縮、一邊 engaged 在漲，這只有一個解釋——首頁把「看一眼就跳出」的人，大量轉成「願意留下來探索」的人。跳出率從接近三分之二砍到三分之一。

而且這不是上線那一天的事。逐日看 AFTER 窗口的 engagementRate：5/29 58% / 5/30 61% / 5/31 67% / 6/1 64% / 6/2 67% / 6/3 63%——整週站穩在 58–67%，沒有一天掉回 BEFORE 的 37%。**新鮮感效應的反證**：D+2 當時擔心 section_view 會從 744/天衰退回 baseline 200，實際 AFTER 穩定落在 450–730/天（spike 日 1500+），沒有衰退。互動是被結構性地建立起來的，不是煙火。

**就算扣掉 spike 日也成立**：5/31 + 6/1 首頁有一波高峰（PV 545 / 568，可能跟 6/1 讀者登入系統上線有關）。把這兩天拿掉，剩下的安靜日（5/29 / 5/30 / 6/2）avgDur 仍 171s、engagementRate 仍 62%，照樣是 BEFORE（63s / 37%）的三倍夯實、近兩倍互動率。結論不靠那兩天撐。

### 新訪客才是真正被改變的人

最關鍵的拆層。把首頁訪客拆成新 / 回訪：

|                           |         每人停留（page 級） | engagementRate |
| ------------------------- | --------------------------: | -------------: |
| **新訪客** BEFORE → AFTER | 51.6s → **177.6s（+244%）** |  38% → **58%** |
| 回訪者 BEFORE → AFTER     |                 171s → 635s |      61% → 64% |

回訪者本來就熟、本來就 engaged（61–64% 兩邊差不多，而且樣本小、635s 那個數字有少數 power user 拉高的雜訊，不要太當真）。**真正被改版改變的是新訪客**——第一次來的人，停留從 52 秒變 178 秒、engaged 比例從 38% 跳到 58%。而新訪客是絕對多數（1,416 vs 196 回訪）。改版把「第一印象」這件事做厚了，這正是首頁最該贏的地方。

（這層拆法也順手排掉一個 confounder：改善不是「因為 AFTER 多了一批本來就 engaged 的回訪者」——是新訪客自己的 engagement 真的三倍了。）

## 四、轉換：率明確變好，但「去哪裡」變了

「轉換」要先定義轉什麼。如果轉換 = 從「跳出」變成「engaged 並往下走」，那答案非常明確：**跳出率砍半、engaged 比例翻倍 = 轉換率明確改善**。

但如果轉換 = 「導去特定目的地頁」，故事就有個必須誠實面對的弔詭。

### 弔詭：首頁互動變強，但 meta 頁的流量卻掉了

| 目的地頁       | BEFORE/天 | AFTER/天 |           Δ |
| -------------- | --------: | -------: | ----------: |
| /about/        |      31.6 |     24.0 |        -24% |
| /semiont/      |      33.7 |     19.8 |    **-41%** |
| /dashboard/    |      11.7 |      9.0 |        -23% |
| /contribute/   |      17.7 |     13.8 |        -22% |
| /explore/      |       9.0 |      8.7 | -4%（持平） |
| /taiwan-shape/ |      16.0 |     26.7 |    **+67%** |

首頁互動明明變強，為什麼 /about、/semiont 這些「介紹型」頁面的流量反而掉了？

把 click 資料攤開就懂了。AFTER 窗口首頁點擊去哪：

| 點擊目標                                    | clicks（5/29-6/4） | 去哪              |
| ------------------------------------------- | -----------------: | ----------------- |
| hero_cta「explore」                         |                205 | /explore          |
| random_dice「roll」                         |                102 | 隨機一篇文章      |
| reader_door「first」                        |                 78 | 精選第一篇文章    |
| category_grid（history/tech/food/culture…） |               124+ | **分類頁 → 文章** |
| reader_door「random」                       |                 51 | 隨機文章          |
| reading_path「step_1」                      |                 27 | 引導閱讀路徑      |

**改版把點擊重新導向了「內容」，而不是「介紹」。** 新的 affordance（分類格、隨機骰子、ReaderDoors、hero explore）全都把人推去**讀文章 / 逛分類**，而不是去 /about 看專案說明、去 /semiont 看概念。meta 頁掉 25–41%，但那是因為讀者被導去了內容本身。

這算好還是壞，取決於首頁的目標。如果目標是「讓人進去讀台灣的東西」，那這是更好的轉換——人去讀文章了，不是去讀「關於我們」。如果目標是「讓人理解這個專案是什麼」，那 /about、/semiont 少了四成的訪客是一個要留意的代價，但這些人很可能改成在首頁上直接看 OrganismPreview + 心跳條（那兩個 section 把器官狀態搬上首頁了，本來要去 /dashboard、/semiont 才看得到）。/taiwan-shape +67% 是唯一逆勢上漲的，對應它在首頁有圖像入口。

## 五、改版後的 on-page 行為（只有 after，但本身就是證據）

這層沒有 before 可比，但它直接證明改版的互動元件「真的被用」，不是裝飾。

### 閱讀時間漏斗（5/29-6/4，唯一用戶）

- ≥30s：660 人
- ≥60s：443 人（30s 的 67%）
- ≥180s：248 人（38%）
- ≥600s：**152 人**（23%）

**整週有 152 個唯一讀者在首頁停滿 10 分鐘。** 每階段留存 67–70% 是健康曲線，過 30 秒篩選的人有清楚閱讀意圖。

### 捲動深度漏斗（5/30-6/4，bug 修好後第一份乾淨資料）

- 捲過 25%：431 人
- 捲到 50%（中段）：347 人（25% 的 81%）
- 捲到 75%：132 人（31%）
- 捲到底 100%：39 人（9%）

捲到中段的留存很強（81%），但下三分之一的 section（contribute / community）只有少數人看到——跟 section 漏斗一致。這份 scroll 資料本身是改版 + 5/29 修 bug 才有的新能力。

### Section 捲動漏斗（誰被看到）

hero 1324 人 → reader_doors 856（65%）→ random_discovery 533（40%）→ feature_cards 352（27%）→ organism_preview 320（24%）→ reading_path 258（19%）→ cover_story 224（17%）→ **category_grid 178（13%）** → … → contribute 68（5%）→ community_feedback 53（4%）。

**一個高槓桿矛盾**：category_grid 是全首頁轉換最強的 section（看到的人幾乎人人點，D+2 量到 CTR 79%），但它埋在第 13% 深度——只有八分之一的讀者捲得到。把分類格往上搬，理論上能放大最會轉換的那個入口。feature_cards 反過來：27% 的人看到它，但幾乎沒人點（D+2 CTR 1.8%），是位置好、affordance 差的死流量，值得重設計。

## 六、誠實的但書（confounder 清單）

把數據講準比把故事講大重要。這份報告的弱點：

1. **流量混入可能性**：AFTER 用戶比 BEFORE 少 26%。如果掉的剛好是低品質 bounce 流量（例如某個導去首頁的低意圖來源在 BEFORE 窗口活躍、AFTER 消失），engagementRate 的上升會有一部分是「壞流量退場」而非「改版變好」。我無法完全排除，但**新訪客 page 級停留 52s→178s 是 per-user 指標、跟流量量無關**，這條繞過了混入問題，是最硬的證據。
2. **instrumentation 只有 after**：on-page funnel 沒有 before 可比，只能證明「改版後互動健康」，不能證明「比改版前好」。
3. **量測變動風險**：改版加了一堆 JS 互動，理論上更互動的頁面會讓瀏覽器 tab 前景時間變長、更容易跨過 GA4 的 10 秒 engaged 門檻。但 engaged 門檻是真的停留時間，不是 custom event（首頁的 section_view 那些不是 key event，不直接灌 engagementRate）。跳出率 + 互動率 + 行為漏斗三者方向一致，支持是真改善而非量測假象。
4. **「停留」的正確讀法**：再強調一次——首頁本身的停留只 +17%，翻四倍的是 session 深度。不要對外講「首頁停留變四倍」，那會在第一個看 GA 的人面前破功。

## 七、Verdict + 建議（待哲宇定）

### 改版十天後的結論

| 維度     | 改善？                   | 證據                                                                                                         |
| -------- | ------------------------ | ------------------------------------------------------------------------------------------------------------ |
| **互動** | ✅ 明確                  | engagementRate 37%→64%、engaged session 數 +26%（用戶反而 -26%）、on-page 元件被持續使用無衰退               |
| **停留** | ⚠️ 要拆著看              | 首頁 page 級停留僅 +17%；但 session 深度 +334%（人被送進站裡讀文章）。「黏在首頁」沒發生，「往裡面走」發生了 |
| **轉換** | ✅ 率明確 / 目的地有取捨 | bounce 砍半 = 轉換率明確變好；但點擊被導向內容（文章/分類），meta 頁（/about -24%、/semiont -41%）流量被吸走 |

**整體**：這是一次成功的改版，而且效果留住了一整週、在新訪客身上最明顯。它把首頁從「門面」改造成「發射台」。唯一要哲宇判斷的取捨是 meta 頁（/about、/semiont）流量被內容入口吸走四成——這是設計選擇的結果，不是 bug。
