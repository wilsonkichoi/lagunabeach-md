## TL;DR

- 轉換器是全站**最老、最工具化的一頁**（2026-03-30 生，種下後第 13 天），現在正在 **SEO 突破期**：搜尋排名 6 週內從 pos 22 爬到 **pos 3.6**，**90 天 175 次點擊裡有 142 次（81%）發生在最近 28 天**。它不是平穩流量，是剛剛起飛。
- **誰想來（SC）**：90d 175 clk / 1,539 impr / **CTR 11.4%**（站平均 1.3% 的 ~9 倍）/ pos 4.5。需求語彙被一條 query 壟斷:**「大陸用語轉換器」845 impr / 94 clk**（占品牌字搜尋的 ~84%）。
- **誰來了（GA）**：90d 轉換器頁 1,163 views / 324 users，**73% 新訪客**、來源約一半 Google organic 一半直連。最近 28d Google organic 已壓過直連成為主引擎。
- **裝置矛盾**：SC（發現）**66% 行動**，GA（停留使用）**偏桌機**。手機上被搜到，桌機上被使用。
- **最大未捕獲面**：2,336 條詞庫**零 per-term 頁**。單詞查找需求（「視頻 台灣用語」「出片台灣用語」「屏蔽台灣用語」pos 1.0）有排名但 **0 點擊**——貼整段的工具回答不了一個詞的問題。

## 一、使用狀況（量得到的那一半）

### 1.1 SEO 突破曲線（這是本報告的主軸）

每週 Search Console，轉換器頁 page-filtered：

| W15–17  |     0–4 |   10–36 |   6.4–22 |      7–26 | 沉寂，排名在第 2–3 頁徘徊     |
| W18     |       7 |      53 |      7.7 |       202 | GA 暴衝（**社群**，非搜尋）   |
| W19     |      12 |      64 |      5.3 |       317 | GA 峰值（Threads referral）   |

讀法：上線（3/30）後排名要爬 6–7 週才從第 3 頁進到第 1 頁。**一旦 mid-May（W20）越過 pos 5，曝光與點擊就 hockey-stick**。這是「園丁模式 > 衝刺模式」的活證據——價值不來自上線那兩週的社群 buzz（W18–19 那兩百多 view 的 GA 暴衝幾乎沒有搜尋點擊，是 Threads 帶的，之後就退潮），來自一條有穩定需求的 query 上慢慢爬。**GA 是誰來了（社群尖峰、會退潮），SC 是誰想來（搜尋慢爬、會複利）**——這頁正在從社群驅動的新奇感，換檔成搜尋驅動的工具。

## 二、深度觀察（量不到、但更重要的那一半）

### 2.1 全站最工具化的一頁，對自己的使用完全失明

轉換器埋了 **0 個互動 event**。對照組是首頁：2026-05-26 哲宇 directive「首頁元素都要加 GA 事件追蹤，徹底了解互動行為」之後，`HomeEventTracker` 埋了 6 種 event（click / outbound / form / section_view / scroll / time）。**那道 directive 從沒延伸到轉換器**——而轉換器比首頁更該埋：它更互動、更工具、SEO 價值更高。

結果是整個操作層全黑：324 個使用者裡，多少人真的貼了字按了轉換？查最多的是哪些詞？用了哪個範例分類（科技/生活/醫療…）？複製率多少？方向 toggle（cn2tw vs tw2cn）誰在用？**全部不知道。** 這是 REFLEXES #15「反覆浮現要儀器化」掉在產品層的版本，也是我這幾天反覆撞的「儀器只看得見存在，看不見缺席」。

更貴的代價是它**掐死一個飛輪**：詞庫該長哪些詞，現在靠作者（哲宇 / contributor）猜，不是靠需求。如果「查了哪個詞」有 event，那條訊號可以直接餵回 YAML 該補什麼、該寫哪篇文。現在這條神經沒接上。

### 2.2 需求的語彙不是工具的語彙（大陸 vs 中國 vs 支語）

品牌字搜尋 84% 是**「大陸用語轉換器」**。工具叫**「台灣用語轉換器」**，頁內 framing 是 中國／台灣。使用者心裡的任務是「把**大陸**話翻回正常」，用的是 PRC 中性偏軟的「大陸」；Taiwan.md 的 identity 用「中國」。第三個人群搜「支語翻譯器」（貶義），又是另一種立場。

這頁現在是**靠巧合**走在鋼索上：它已經 rank 在「大陸用語轉換器」pos 3.9（接住了需求），頁內又守著 中國／台灣 的 framing（守著 identity）。但 SEO title 與 H1 都以「台灣用語」開頭，等於在最大那條需求字上少了一塊對齊。這是 sovereignty-lens 的張力：**meet-them-where-they-search 跟 hold-your-frame 可以並存**，但要刻意設計，不能靠運氣。名字本身在做 identity 的工——不管你有沒有意識到。

### 2.3 兩種 intent 共用一頁，而且贏的是錯的那種

- **「轉換我這段」**：轉換器的格式。會點（CTR 11–30%）。
- **「X 這個詞台灣怎麼說」**：單詞查找。**不點**（CTR 0%）。

「視頻 台灣用語」「出片台灣用語」「打碟台灣用語」「屏蔽台灣用語」（pos 1.0 還是 0 點擊）——它們 rank 得到、但點不下去，因為一個「貼整段」工具的 SERP 摘要回答不了「視頻在台灣叫什麼」。**2,336 條詞庫，0 個 per-term 頁。** 這是全站最大的未捕獲 SEO 面：每一條詞都可以是一頁，直接回答「X 台灣怎麼說」+ 詞源（YAML 已經有 fork_type / etymology / sources，料是現成的）。這是跟轉換器讀者完全不同的人——一個好奇一個詞，一個在清整篇草稿。

### 2.4 它正在 SEO 突破，而會複利的是搜尋不是社群

§一.1 的曲線已經講完：別再推一次社群。W18–19 的社群暴衝退潮了，W20+ 的搜尋慢爬在複利。要投資的是搜尋面（per-term 頁、schema、大陸用語 對齊的 title），不是下一支 Threads。

### 2.5 它其實是一座主權巴別塔，但我們看不見有沒有人往回走

中國 47 / 香港 9 / 新加坡 18 / 美國 50 的離岸使用者。tw2cn（台灣→中國）反向就是為這群人造的——在中國想讀寫台灣華語的人，或要寫給對岸看的台灣人。轉換器安靜地是個 §主權的巴別塔 artifact：讓台灣的第一人稱語感跨海峽存在。但**反向有沒有人走，完全不可見**（無 event，方向只存 localStorage）。這個 mission frame 比「幫台灣人清支語」大得多，而我們連它活不活著都不知道。

## 三、怎麼進化（提案，交哲宇拍板）

標記：✅ = Semiont 可自決 / 🔒 = 需哲宇拍板（命中 §自主權邊界）。

| ID     | 提案                                                                                                                                                                       | 價值                                               | Effort                  | 自主權                              |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------- | ----------------------- | ----------------------------------- |
| CONV-1 | **互動埋點**（mirror HomeEventTracker）：`convert` / `example_click{category}` / `copy` / `direction_toggle{dir}` / **`term_lookup{rule_id}`**（每條 matched rule 發一次） | 點亮整個操作層；`term_lookup` 是飛輪的種子         | 小–中（1 檔）           | ✅                                  |
| CONV-2 | **Per-term 靜態頁** `/terminology/[id]`：2,336 YAML → 頁，直接答「X 台灣怎麼說」+ 詞源 + 反查連回轉換器 + schema                                                           | 接住單詞 long-tail（§二.3）；全站最大未捕獲 SEO 面 | 中（新路由 + 2,336 頁） | 🔒（>50 檔 + SEO 方向）             |
| CONV-3 | **SEO 對齊「大陸用語」intent**：title / JSON-LD `alternateName` / 一條 FAQ 納入「大陸用語轉換器」，**頁內 中國／台灣 framing 不動**                                        | 對齊 84% 搜尋語彙、同時守 identity（§二.2）        | 小                      | 🔒（碰兩岸用詞，政治立場 adjacent） |
| CONV-4 | **Mobile-first UX**：單欄時結果即時浮現 + sticky 一鍵複製、收窄 hero、範例 chip 改可橫滑                                                                                   | 接住 66% 行動發現流量的第一印象（§一.4）           | 中                      | ✅                                  |
| CONV-5 | **demand → term-DB 飛輪**：`term_lookup` event 聚合 → 哪些詞被查最多但詞庫沒有 / 待補 → 排進 ARTICLE/詞庫優先序                                                            | 詞庫成長從 author-guess 變 demand-driven（§二.1）  | 中（依賴 CONV-1）       | ✅                                  |
| CONV-6 | **反向方向先量再說**：CONV-1 的 `direction_toggle` 上線後，量 tw2cn 真實使用率，再決定強化 / 收掉                                                                          | 證據先於設計，別猜（§二.5）                        | 小（含在 CONV-1）       | ✅                                  |

**建議順序**：CONV-1（先點亮，所有其他提案的前提）→ CONV-3（小、SEO ROI 最高，但等哲宇用詞拍板）→ CONV-2（最大 SEO 面，但最大 effort + 方向決策）→ CONV-4 → CONV-5/6（資料累積後）。

**為什麼 CONV-1 排第一**：現在每個進化決策都在盲量。先把操作層點亮，CONV-2/4/5 的成效才有得驗（per REFLEXES #67，沒有 runtime 量測就只能讀碼推論）。CONV-1 是 1 檔、純 tooling、Semiont 可自決——是這份報告唯一我想先動手的。
